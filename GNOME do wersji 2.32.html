[edytuj] Wprowadzenie

Środowisko graficzne GNOME (ang. GNU Network Object Model Environment) jest częścią projektu GNU i zostało wydane na licencji GPL. Projekt został rozpoczęty w sierpniu 1997 roku przez Miguela de Icaza w celu stworzenia w pełni wolnodostępnego graficznego środowiska użytkownika dla systemu GNU/Linux, jako alternatywy dla środowiska KDE, opartego o bibliotekę Qt (do listopada 1997 nie będącą wolnym oprogramowaniem). Środowisko GNOME składa się wyłącznie z wolnego i otwartego oprogramowania.

Projekt GNOME dostarcza dwóch rzeczy: środowiska graficznego GNOME, intuicyjnego i atrakcyjnego pulpitu dla użytkowników, oraz platformy programistycznej GNOME, czyli rozbudowanego zestawu narzędzi do tworzenia aplikacji zintegrowanych z tym środowiskiem.

Projekt GNOME kładzie nacisk na prostotę i użyteczność. Inne cele projektu:

  • Wolność do tworzenia środowiska graficznego, którego kod źródłowy będzie zawsze dostępny.
  • Zapewnienie pulpitu dla każdego, niezależnie od umiejętności technicznych czy stopnia niepełnosprawności.
  • Internacjonalizacja, czyli dostępność GNOME w wielu językach. W tej chwili GNOME jest przetłumaczone w 166 językach.
  • Zapewnienie przyjaznej pracy deweloperom, niezależnie od używanego języka programowania.
  • Organizacja regularnych cykli wydawniczych i zdyscyplinowana struktura wspólnoty.
  • Zapewnienie wsparcia ze strony innych instytucji.

[edytuj] Instalacja

Upewnij się, że masz zainstalowaną grupę xorg oraz jej zależności:

# pacman -S --needed xorg

Aby zainstalować środowisko graficzne GNOME, instalujemy grupę gnome wydając polecenie:

# pacman -S gnome

Oto lista pakietów z grupy gnome, które mogą być niepotrzebne:

  • epiphany - domyślna przeglądarka internetowa środowiska GNOME;
  • gnome-backgrounds - zbiór tapet;
  • gnome-screensaver - kolekcja wygaszaczy;
  • gnome-themes - zbiór kompozycji pulpitu;
  • gnome2-user-docs - dokumentacja (często w języku angielskim)
  • yelp - dokumentacja (często w języku angielskim)
  • libgail-gnome - biblioteka wykorzystywana np. przez czytnik ekranu Orca
  • yelp - pomoc GNOME

Dodatkowe pakiety, zwiększające funkcjonalność środowiska, znajdują się w grupie gnome-extra:

# pacman -S gnome-extra

Oto lista pakietów z grupy gnome-extra, które mogą być niepotrzebne:

  • bug-buddy - pakiet służący do raportowania błędów.
  • cheese
  • dasher - program, za pomocą którego można wprowadzać znaki wskaźnikiem, nie używając klawiatury.
  • deskbar-applet - pasek wyszukiwania dla GNOME
  • ekiga - wideo-komunikator
  • empathy - komunikator
  • evolution, evolution-exchange oraz evolution-webcal - kalendarz sieciowy i klient pocztowy
  • file-roller - archiwizator
  • gcalctool - kalkulator
  • gedit - edytor tekstu. Jeżeli używasz innego edytora, nie musisz instalować tego
  • gnome-audio - kolekcja dźwięków dla GNOME
  • gnome-games i gnome-games-extra-data - kolekcja prostych gier
  • gnome-mag - lupa ekranowa
  • gnome-nettool i gnome-netstatus - zbiory klikalnych narzędzi sieciowych
  • gnome-utils - zbiór narzędzi dla GNOME
  • gucharmap - tablica znaków Unicode
  • gok - klawiatura ekranowa
  • hamster-applet
  • mousetweaks - niestandardowe ustawienia myszki
  • orca - czytnik ekranu
  • seahorse i seahorse-plugins - menadżer haseł i kluczy szyfrujących (to warto mieć)
  • sound-juicer - zgrywarka płyt cd-audio
  • tomboy - menadżer notek pulpitowych
  • totem - odtwarzacz multimedialny
  • vinagre - klient VNC
  • vino - pakiet do zarządzania zdalnym serwerem
  • zenity - proste okna dialogowe
Notka: Pakiet gnome-system-tools nie znajduje się już w grupie gnome-extra. Trzeba go oddzielnie doinstalować. Wydaj w tym celu polecenie:
# pacman -S gnome-system-tools gksu

[edytuj] Uruchomienie GNOME

GNOME z uruchomimy z konsoli za pomocą polecenia:

$ xinit gnome-session

Dodajemy poniższy wpis dla konkretnego użytkownika ~ /.xinitrc (i upewnij się, że jest to jedyna linia, która zaczyna się od exec):

File: ~/.xinitrc
exec gnome-session

W przypadku problemów z automontowaniem:

File: ~/.xinitrc
exec ck-launch-session gnome-session

Aby ustawić globalnie dla wszystkich użytkowników, dodajemy wpis do pliku /etc/X11/xinit/xinitrc.

[edytuj] GNOME Display Manager

GDM (GNOME Display Manager) to menedżer logowania dla środowiska graficznego GNOME. Został zaprojektowany jako prostsza w konfiguracji alternatywa dla domyślnego w X11 menedżera XDM.

# pacman -S gdm

a następnie na końcu sekcji DAEMONS w pliku /etc/rc.conf dodajemy odpowiedni wpis:

File: /etc/rc.conf
DAEMONS=(...gdm)

Jeżeli chcesz przeczytać więcej GDM, odwiedź artykuł.

[edytuj] Zalecane demony i moduły

  • D-Bus

Na początek instalujemy demona:

# pacman -S dbus

Następnie edytujemy plik /etc/rc.conf:

File: /etc/rc.conf
DAEMONS = (...dbus...)

Na koniec ręcznie uruchomiamy DBUS wpisując w linii poleceń:

# /etc/rc.d/dbus start

Aby w pełni wykorzystać możliwości DBUS w swoim systemie, użytkownik musi należeć do wymaganych grup, a zatem:

# gpasswd -a nazwa_użytkownika optical # gpasswd -a nazwa_użytkownika storage # gpasswd -a nazwa_użytkownika dbus # gpasswd -a nazwa_użytkownika disk

Jeżeli włączyliśmy DBUS przed dopisaniem użytkownika do grup, należy zrestartować demona przy pomocy poleceń:

# /etc/rc.d/dbus restart
  • GVFS

Aby korzystać z możliwości montowania wirtualnych systemów plików, dodaj moduł fuse do pliku /etc/rc.conf:

File: /etc/rc.conf
MODULES= (...fuse...)

[edytuj] Obserwator plików

Jest to proces który informuje nasze programy o zmianach, które nastąpiły na dysku (bez tego GNOME działa jak upośledzone).

[edytuj] FAM

Instalacja FAM:

# pacman -S fam

Następnie na końcu sekcji DAEMONS w pliku /etc/rc.conf dodajemy wpis fam np:

File: /etc/rc.conf
DAEMONS=(...fam gdm)

Następnie:

# /etc/rc.d/fam start

[edytuj] Gamin

Gamin jest alternatywą dla pakietu FAM, działa jako biblioteka menadżera plików. Aby zastąpić FAM, wystarczy go usunąć, ignorując zależności i zainstalować Gamin.

# pacman -Rd fam # pacman -Sy gamin

Gamin nie działa jako daemon, nie wymaga wpisu w /etc/rc.conf.

[edytuj] Konfiguracja GNOME

[edytuj] Dodawanie lub edycja sesji GDM

Każda sesja to plik *.desktop, który znajduje się /usr/share/xsessions.

Aby dodać nową sesję, skopiuj istniejący plik *.desktop do wykorzystania jako szablon dla nowej sesji:

$ cd /usr/share/xsessions # cp gnome.desktop other.desktop # sudo nano other.desktop

[edytuj] Zmiana menedżera okien

Domyślnym menedżerem okien w GNOME jest Metacity. Jednakże możemy wybrać sobie alternatywę dla niego. Poniżej opisany jest przykład z Openboksem:

  1. Zaloguj się do swojej sesji GNOME (która powinna dalej używać Metacity jako menadżera okien) jeśli nie jesteś jeszcze zalogowany.
  2. Zainstaluj Openboksa, jeśli jescze tego nie zrobiłeś. (pacman -S openbox obconf)
  3. Przejdź do swojego menu System > Preferencje > Programy Startowe.
  4. W otworzonym oknie naciśnij Dodaj i wpisz tekst wyświetlony poniżej.
  5. Teraz wciśnij przycisk Dodaj w oknie z wprowadzonymi informacjami i upewnij się, że pole wyboru obok nowego rekordu jest zaznaczone.
  6. Wyloguj się ze swojej sesji GNOME i zaloguj ponownie. Od teraz, powinieneś używać Openboksa jako swojego menadżera okien. 
Nazwa:    Openbox Windox Manager
Polecenie: openbox --replace
Komentarz: Zastępuje metacity Openboksem

Poinformuj jeszcze tylko GNOME o Openboksie:

$ gconftool-2 -s -t string /desktop/gnome/session/required_components/windowmanager openbox

[edytuj] Blokada ekranu

Upewnij się, że posiadasz wpis dbus w sekcji DAEMONS w pliku /etc/rc.conf oraz że masz zainstalowany jeden z pakietów:

# pacman -S {{Package Official|xscreensaver}} # pacman -S {{Package Official|gnome-screensaver}}

[edytuj] Brak ikon w Menu

# gconftool-2 --toggle /desktop/gnome/interface/menus_have_icons # gconftool-2 --set --type boolean /desktop/gnome/interface/menus_have_icons true

[edytuj] Brak ikon w przyciskach

# gconftool-2 --toggle /desktop/gnome/interface/buttons_have_icons

[edytuj] Brak możliwości wybrania motywu kursora

Tworzymy plik /usr/share/icons/default/index.theme o zawartości:

File: /usr/share/icons/default/index.theme
[icon theme] 
Inherits=nazwa_motywu

Tworzymy plik /home/nazwa_użytkownika/.Xdefaults o zawartości:

File: /home/nazwa_użytkownika/.Xdefaults
Xcursor.theme: nazwa_motywu

Teraz wystarczy wylogować się.

[edytuj] Czcionki

Możesz zmienić DPI czcionek w GNOME - naciśnij prawym przyciskiem myszy na pulpicie. Ustawienia znajdziesz w zakładce "Czcionki". Wartość DPI ustaw na 96.

[edytuj] Domyślna tapeta

Domyślnym tłem w GNOME jest zielony listek. Obraz jest wyświetlany, na przykład gdy ekran jest zablokowany. Jeżeli chcemy ustawić własną tapetę jaką domyślną, musimy podmienić ją (z zachowaniem nazwy) w tym miejscu: 

/usr/share/pixmaps/backgrounds/gnome/background-default.jpg

[edytuj] GNOME jest powolne

Sprawdź, czy /etc/hosts masz wiersz podobny do poniższego. Jeżeli nie, wpisz tak, jak pokazuje przykład:

File: /etc/hosts
127.0.0.1       localhost.localdomain     localhost nazwa_użytkownika

Drugim powodem słabej wydajności GNOME może być wykorzystany motyw ikon. Jeżeli ikony są w formacie SVG, mogą spowolnić system, dlatego też powinna być poprawa przy użyciu ikon w formacie PNG.

[edytuj] Ikony na pulpicie

Otwórz terminal, następnie wpisz:

$ gconf-editor

Włączanie/wyłączanie ikon na pulpicie znajdziesz w: 

apps > nautilus > desktop

[edytuj] Katalogi użytkownika

W wielu dystrybucjach, takich jak Ubuntu czy Linux Mint, skonfigurowane są domyślne katalogi z przeznaczeniem do przechowywania ściągniętych z sieci plików, muzyki, dokumentów... Każdy z nich zawiera również specjalną ikonę identyfikacji. Aby ustawić XDG do katalogów użytkownika, należy uruchomić polecenie:

# pacman -S xdg-user-dirs

Konfiguracja XDG zawarta jest w pliku /etc/xdg/user-dirs.defaults lub w ~/.config/user-dirs.dirs dla konkretnego konta. Aby odświeżyć ustawienia, wystarczy wydać polecenie:

$ xdg-user-dirs-update

[edytuj] Konfiguracja szybkości Menu

Możesz usunąć opóźnienia w menu GNOME, uruchamiając polecenie:

$ echo "gtk-menu-popup-delay = 0" >> ~/.gtkrc-2.0

[edytuj] Zmiana ikony Menu

Ikona, wykorzystana w Menu zawsze nosi nazwę: 

start-here.png
start-here.svg

Możesz znaleźć ją w dwóch miejscach:

  • Motyw domyślny w GNOME 
/usr/share/icons/gnome/24x24/places/
  • W przypadku wykorzystanego własnego motywu ikon 
/home/nazwa_użytkownika/.icons/(tu mogą być inne katalogi, zależnie od motywu)/places/

Teraz wystarczy zrestarować panel GNOME:

$ pkill gnome-panel

[edytuj] Zmiana rozmiarów terminala

W tym pomoże polecenie:

$ gnome-terminal --geometry 105x25+100+20

Aby wprowadzić swoje ustawienia na stałe, wyedytuj plik /usr/share/vte/termcap/xterm. Poszukaj wiersza co#80:it#8:li#24:\. Za szerokość okna terminala odpowiada pierwsza wartość, za wysokość trzecia.

[edytuj] Ustawienia zegara w panelu

$ gconf-editor
/->apps->panel->applets->clock_screen->prefs

Przykłady:

  • <span size="smaller">%a %d %b</span> <b>%H:%M</b>
  • <b>%H:%M</b> <span size="smaller">%x</span>

Na koniec przechodzimy w tym samym katalogu do wpisu format i nadajemy mu wartość custom.

[edytuj] Zmiana rozmiarów okna Nautilusa

Poważną wadą Nautilusa jest to, że rozmiary jego oknien zawsze wracają do domyślnych wartości. Jeżeli chcesz sam ustawić sobie domyślne rozmiary okna Nautilusa, wykonaj poniższe czynności.

Wpisujemy w terminalu:

$ gconf-editor

Następnie przechodzimy do folderu: 

/->apps->nautilus->preferences

Tworzymy klucz tekstowy o nazwie: 

navigation_window_saved_geometry

i wartości przykładowej: 

649x338+94+67

[edytuj] Własne skrypty w prawokliku

Wyjasnienie opiera się na przykładzie. Przypuśćmy, że chcemy mieć mozliwość uruchomienia w prawokliku pliku konfiguracyjnego /etc/rc.conf. Tworzymy więc skrypt z zawartością:

File: Skrypt
#!/bin/sh
gnome-terminal -e 'sudo gedit /etc/rc.conf'

Następnie umieszczamy go w /home/nazwa_użytkownika/.gnome2/nautilus-scripts/. I to wszystko. Pamiętajmy, że polecenie sudo daje nam spore uprawnienia.

To jest tylko przykład. Możemy również tworzyć skrypty, które pozwolą wykonać nam wszystko, co tylko robimy w naszym terminalu.

[edytuj] Rozwiązywanie problemów

[edytuj] Główne rozwiązanie

Jeżeli nie wiesz, co jest przyczyną niepoprawnie działającego GNOME, spróbuj przenieść jego konfigurację w inne miejsce:

$ for d in .gnome* .gconf*; do mv "$d" "$d.old"; done

Teraz, używając nowych plików konfiguracyjnych, sprawdź środowisko restartując je.

[edytuj] Po awarii komputera GNOME nie chce załadować się

Zastosuj polecenie:

$ mv ~/.gnome2/session ~/.gnome2/session.old

[edytuj] Ekran jest czarny/ciemny podczas ładowania GNOME

W konsoli wpisz:

$ gconf-editor

Znajdź / → apps → gnome-power-manager → backlight i zmień wartość brightness_ac z 100 na 0.

[edytuj] Niepoprawne znaki w nazwach plików na partycjach FAT

Poniższe polecenie zlikwiduje problem; Arch będzie odczytywał i zapisywał nazwy z Twoim wyborem językowym:

$ gconftool-2 -s /system/storage/default_options/vfat/mount_options --list-type=string -t list [shortname=lower,uid=,utf8]

[edytuj] gnome-terminal nie wspiera UTF-8

Dodaj poniższe wiersze do pliku /etc/environment: 

LANG="pl_PL.UTF-8"
LC_ALL="pl_PL.UTF-8"

Zrestartuj system.

[edytuj] Brak paneli podczas uruchamiania sesji

Jeżeli zdarzy się, że usunąłeś wszystkie panele, bądź powstał problem z ich uruchomieniem, użyj poniższego skryptu (wymagany jest pakiet zenity): 

#!/bin/sh
#
# GNOME Panel Save / Restore
# Writen by PhrankDaChicken
#
# http://ubuntu.online02.com
#
#
# Updated to add restore defaults by jimjimovich
# http://www.starryhope.com
#
#


DIR=$(pwd)
TITLE="PanelRestore"

Main () {
	CHOICE=$(zenity --list --title "$TITLE" --hide-column 1 --text "What do you want to do?" --column "" --column "" \
"0" "Save Panel Settings" \
"1" "Restore Panel Settings" \
"2" "Restore Default Panel Settings")
	if [ $CHOICE = 0 ]; then
		Panel_Save
	fi
	if [ $CHOICE = 1 ]; then
		Panel_Restore
	fi
	if [ $CHOICE = 2 ]; then
		Panel_Defaults
	fi	
}

Panel_Restore () {
	FILE=$(zenity --title "$TITLE: Open File" --file-selection --file-filter "*.xml" )
	if [ -n "$FILE" ]; then 
		gconftool-2 --load "$FILE"
		killall gnome-panel
	fi
	Main
}

Panel_Save () {
	FILE=$(zenity --title "$TITLE: Save File" --file-selection --save --confirm-overwrite --filename "Gnome_Panel.xml" --file-filter "*.xml" )
	if [ -n "$FILE" ]; then 
		EXT=$(echo "$FILE" | grep "xml")
		if [ "$EXT" = "" ]; then
			FILE="$FILE.xml"
		fi
		gconftool-2 --dump /apps/panel > $FILE
		zenity --info --title "$TITLE: File Saved" --text "File saved as: \n $FILE"
	fi
	Main
}

Panel_Defaults () {
	zenity --question --text="Are you sure you want to restore the default top and bottom panels?"
	gconftool-2 --recursive-unset /apps/panel
	rm -rf ~/.gconf/apps/panel
	pkill gnome-panel
	exit
}

Main
