[edytuj] Opis

Wiele nowych procesorów obsługuje dynamiczną zmianę taktowania w trakcie pracy, dostosowując się do aktualnego obciążenia, co zapobiega nadmiernemu nagrzewaniu się procesora oraz zużyciu energii w czasie, gdy duża moc nie jest do niczego potrzebna. Rozwiązanie to szczególnie sprawdza się w notebookach podczas pracy na zasilaniu bateryjnym. W tym artykule pokażemy, jak skonfigurować Cpufreq, który pozwoli osiągnąć ten efekt w Arch Linuksie.

<a name="Instalacja"></a>

Instalacja

# pacman -S cpufrequtils
<a name="Konfiguracja"></a>

Konfiguracja

Cpufreq pozwala na wybranie polityki regulowania częstotliwości poprzez tzw. zarządców (ang. governors). Każdy z nich realizuje to zadanie inaczej i dlatego musimy wybrać takiego, który najbardziej nam odpowiada. Proces konfigurowania składa się z trzech etapów:

  • Załadowanie odpowiedniego sterownika regulacyjnego
  • Załadowanie zarządcy
  • Konfiguracja i uruchomienie demona sterującego
<a name="Sterownik_regulowania_cz.C4.99stotliwo.C5.9Bci_CPU"></a>

Sterownik regulowania częstotliwości CPU

Do prawidłowego odczytu parametrów procesora i zarządzania nimi Linux musi znać możliwości Twojego sprzętu. Realizowane jest to przy pomocy odpowiedniego sterownika jądra.

<a name="Intel"></a>

Intel

# modprobe acpi-cpufreq

W przypadku starszych procesorów Intela może zaistnieć taki problem: 

FATAL: Error inserting acpi_cpufreq ([...]/acpi-cpufreq.ko): No such device

W tym przypadku trzeba zastąpić acpi-cpufreq na speedstep-centrino / p4-clockmod / speedstep-ich.

Aby sterownik ładował się automatycznie przy starcie, trzeba dodać go do sekcji MODULES w /etc/rc.conf: 

MODULES=( acpi-cpufreq ... )
<a name="AMD"></a>

AMD

# modprobe powernow-k{6,7,8}

Aby sterownik ładował się automatycznie przy starcie, trzeba dodać go do sekcji MODULES w /etc/rc.conf: 

MODULES=( powernow-k{6,7,8} ... )

Przykład: 

# modprobe powernow-k8

W sekcji MODULES w /etc/rc.conf: 

MODULES=( powernow-k8 ... )
<a name="Cpufreq-info"></a>

Cpufreq-info

Aby dowiedzieć się szczegółowych informacji o procesorze: 

# cpufreq-info

Przykładowe wyjście cpufreq-info (z procesora Intel Duo Core T2500):

driver: acpi-cpufreq
CPUs which need to switch frequency at the same time: 0 1
hardware limits: 1000 MHz - 2.00 GHz
available frequency steps: 2.00 GHz, 1.67 GHz, 1.33 GHz, 1000 MHz
available cpufreq governors: ondemand, performance
current policy: frequency should be within 1000 MHz and 2.00 GHz.
                The governor "performance" may decide which speed to use
                within this range.

analyzing CPU 1:
driver: acpi-cpufreq
CPUs which need to switch frequency at the same time: 0 1
hardware limits: 1000 MHz - 2.00 GHz
available frequency steps: 2.00 GHz, 1.67 GHz, 1.33 GHz, 1000 MHz
available cpufreq governors: ondemand, performance

                The governor "performance" may decide which speed to use
                within this range.

Zwróć szczególną uwagę na hardware limits oraz available frequency steps - są to zakresy pracy obsługiwane przez procesor.

<a name="Zarz.C4.85dcy"></a>

Zarządcy

Zarządcy ładowani są jako sterowniki jądra, dzięki czemu mogą być widoczni przez zewnętrzne aplikacje. Możesz mieć załadowanych jednocześnie wielu zarządców, ale aktywny w danej chwili może być tylko jeden.

Dostępni zarządcy:

  1. performance (domyślny) -- wbudowany w jądro zarządca, w którym procesor pracuje zawsze z maksymalną prędkością.
  2. cpufreq_ondemand (zalecany) -- dynamicznie zmienia taktowanie zegara, bazując na aktualnym obciążeniu systemu.
  3. cpufreq_conservative -- podobny do "ondemand", lecz bardziej konserwatywny.
  4. cpufreq_powersave -- procesor pracuje z minimalną prędkością.
  5. cpufreq_userspace -- taktowanie jest zmieniane manualnie przez użytkownika.

Dodaj interesujących Cię zarządców do /etc/rc.conf: 

 MODULES=(cpufreq_ondemand cpufreq_powersave ... )

Można to także zrobić ręcznie komendą cpufreq-set (jako root), lecz nie zostanie to odtworzone po ponownym uruchomieniu komputera. Przykład: 

# cpufreq-set -g ondemand

Aby uzyskać więcej informacji, uruchom 

# cpufreq-set --help
# man cpufreq-set
<a name="Demon"></a>

Demon

Pakiet cpufrequtils instaluje również demona, który umożliwia ustawienie odpowiedniego zarządcy oraz żądanych limitów pracy w trakcie uruchomienia systemu bez potrzeby użycia dodatkowych narzędzi. Jest to najlepsze rozwiązanie dla osób używających lekkich środowisk graficznych, np. Openboksa.

Przed uruchomieniem demona, edytuj jako root plik /etc/conf.d/cpufreq, ustawiając odpowiedniego zarządcę i wybierając minimalną/maksymalną prędkość Twojego CPU: 

#configuration for cpufreq control
# valid governors:
#  ondemand, performance, powersave,
#  conservative, userspace
governor="ondemand"

# valid suffixes: Hz, kHz (default), MHz, GHz, THz
min_freq="1GHz"
max_freq="2GHz"
<a href="/wiki/Grafika:Info.png" class="image" title="Porada"><img alt="Porada" src="/images/Info.png" width="40" height="40" border="0" /></a> Dokładne wartości taktowania znajdziesz po uruchomieniu komendy cpufreq-info dostępnej po załadowaniu sterownika sterującego. Wartości te są jednak opcjonalne. Możesz pominąć je całkowicie poprzez skasowanie lub zakomentowanie linijek min/max_freq. Demon wybierze wtedy zakres automatycznie.


Możesz teraz uruchomić demona: 

# /etc/rc.d/cpufreq start

Nie zapomnij dodać go do tablicy DAEMONS w /etc/rc.conf tak, aby uruchamiał się automatycznie przy starcie: 

DAEMONS=(syslog-ng hal <b>cpufreq</b> ...)
