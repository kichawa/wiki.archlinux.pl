[edytuj] Wprowadzenie

Właściciele kart graficznych ATI mają wybór sterownika pomiędzy własnościowym (catalyst), a otwartym (xf86-video-ati). W tym artykule skupmy się jednak na tym drugim.

Obecnie wydajność sterowników otwarto źródłowych nie jest na równi z odpowiednikiem własnościowym, są mniej wydajne w grafice 3D oraz brakuje im niektórych funkcji, takich jak rzetelne wsparcie TV-out. Jeżeli nie jesteś pewien, którego sterownika wolisz użyć, spróbuj najpierw otwartego - generalnie powinno być z nim mniej problemów.


Sterowniki xf86-video-ati:

  • Współpracują z chipsetami do wersji HD4XXX (najnowsze chipsety R700), jak również HD5XXX (najnowsze chipsety R800).
  • Radeony do serii X1XX są w pełni obsługiwane, stabilne oraz posiadają pełne wsparcie 2D i 3D.
  • Radeony od serii HD2XXX (i nowsze) mają wsparcie 2D i 3D, ale nie wszystkie funkcje są w pełni obsługiwane.
  • Wszystkie karty z serii HD5XXX (R8XX) i nowsze są obsługiwane, ale obecnie tylko z akceleracją 2D.
Notka: Pełna lista wspieranych modeli dostępna jest tutaj, zaś aktualny stan rozwoju sterowników można obserwować na stronie X.Org Fundation.

[edytuj] Instalacja

Uwaga!
Jeśli wcześniej instalowałeś sterowniki zamknięte, upewnij się, że usunąłeś je ze swojego systemu.

Aby zainstalować xf86-video-ati, wydaj polecenie:

# pacman -S xf86-video-ati libgl ati-dri

W przypadku Archa x86_64 warto zainstalować także pakiet lib32-ati-dri, który może być wymagany przez niektóre gry.

W przypadku kart od wersji HD3XXX (R600) zalecane jest zainstalowanie linux-firmware:

# pacman -S linux-firmware

Za nieoficjalne repozytoria deweloperzy Arch Linuksa nie ponoszą odpowiedzialności.
File: /etc/pacman.conf
[radeon]
Server = http://spiralinear.org/perry3d/i686

[radeon]
Server = http://spiralinear.org/perry3d/x86_64

[edytuj] Konfiguracja

Obecnie cała konfiguracja sterowników graficznych odbywa się bez używania pliku /etc/X11/xorg.conf. Jeżeli chcesz mieć wpływ na konfigurację, użyj /etc/X11/xorg.conf.d/20-radeon.conf, który powinien być jedynie opcjonalny bądź używany w przypadku problemów z wykrywaniem sprzętu.

Jeżeli zdecydowałeś się sam skonfigurowaś sobie ustawienia graficzne, zacznij od utworzenia pliku /etc/X11/xorg.conf.d/20-radeon.conf o poniższej zawartości:

File: /etc/X11/xorg.conf.d/20-radeon.conf
Section "Device"
    Identifier "Tu_wpisz_alias"            # ustaw alias z obecnej konfiguracji
    Driver "radeon"
EndSection
Ostrzeżenie: Poniżej przedstawiona jest konfiguracja pliku /etc/X11/xorg.conf, choć jak już zostało to wspomniane powyżej, jest on już nie używany przez obecną wersję Xorga.

Jeżeli jednak używasz pliku /etc/X11/xorg.conf, sprawdź, czy poniższe sekcje wyglądają następująco:

File: /etc/X11/xorg.conf
Section "Module"
 Load  "glx"
 Load  "dri"
 Load  "drm"
EndSection
  
Section "Device"
 Identifier "nazwa"                     # ustaw alias z obecnej konfiguracji
 Driver "radeon"
 Option "XAANoOffscreenPixmaps" "true" # wymagane dla aiglx
EndSection

W celu poprawy wydajności sterownika xf86-video-ati można umieścić poniższe opcje w sekcji "Device" w pliku /etc/X11/xorg.conf:

File: /etc/X11/xorg.conf
Section "Device"
 Option "ColorTiling" "on"     # Odblokowanie tej opcji jest całkowicie bezpieczne.
 Option "AccelMethod" "EXA"    # Opcja działa tylko na nowszych kartach.
 Option "EXAVSync" "on"        # EXAVSync jest powiązana z powyższą opcją.
 Option "DMAForXv" "on"
 Option "EnablePageFlip" "on"  # Odblokowanie Page Flip zwykle jest bezpieczne. Zalecane przy starszych kartach. Odblokowanie tej opcji wyłącza EXA.
 Option "RenderAccel" "on"
 Option "AccelDFS" "on"
 Option "DRI" "on"
 Option "AGPMode" "4"          # Domyślnie sterowniki działają z szybkością 1x AGP. Z reguły bezpiecznie jest zwiększyć szybkość do 4x.
 Option "ScalerWidth" "2048"
 Option "DynamicPM" "on"       # Zarządzanie zasilaniem jest dynamiczne.
 Option "ClockGating" "on"     # Wspomaganie wyboru oszczędzania energii.
EndSection

[edytuj] Kernel Mode Setting (KMS)

Dzięki tej technologii kernel ma znacznie większą kontrolę nad układami graficznymi i przy włączonym KMS może ustawić odpowiedni tryb graficzny dla danego ekranu, gdy tylko zainicjalizowane zostaną wszystkie potrzebne komponenty sprzętowe (ACPI, PCI, układ graficzny itd.).

Z punktu widzenia użytkownika nie różni się to na pierwszy rzut oka zbytnio od grafiki bufora ramki z zastosowaniem odpowiedniego sterownika – jednak w przeciwieństwie do tego sposobu inicjalizacji, w przypadku zastosowania mechanizmu KMS jądro dba o odpowiednie ustawienie rozdzielczości ekranu również w dalszej fazie działania systemu, zwalniając z tego zadania X serwer. Dopóki X serwer oraz konsola tekstowa wspomagana przez KMS używają tej samej rozdzielczości ekranu, kernel nie musi na nowo ustawiać parametrów układu graficznego oraz rozdzielczości ekranu przy przełączaniu między powłoką graficzną a konsolą.

Notka: Kernel Mode Setting, wraz z wydaniem jądra 2.6.33, jest domyślnie włączony.
Notka: Wszystkie pakiety, wymagane do poprawnego działania akceleracji 3D, są wydane jako stabilne i znajdują się w repozytoriach. Mimo wszystko zaleca się zainstalowanie pakietu radeon_ucode.

[edytuj] Wczesny start

1) Usuwamy wszystkie wpisy "vga=" i "video=" z /boot/grub/menu.lst (GRUB) lub /boot/grub/grub.cfg (GRUB2). Używanie innych buforów ramek, takich jak uvesafb/radeonfb, wywoła konflikt z KMS. Dlatego usuń wszystkie z MODULES w pliku /etc/mkinitcpio.conf.

2) Dodajemy "radeon" do MODULES w pliku /etc/mkinitcpio.conf.

3) Dodajemy "fbcon" do MODULES w pliku /etc/mkinitcpio.conf.

Następnie wydajemy polecenie:

# mkinitcpio -p kernel26

4) Dopisujemy "radeon.modeset=1" do /boot/grub/menu.lst (GRUB) lub /boot/grub/grub.cfg (GRUB2).

Przykład dla GRUB: 

kernel /boot/vmlinuz26 root=/dev/sda3 ro radeon.modeset=1

Przykład dla GRUB2: 

linux /boot/vmlinuz26 root=/dev/sda1 ro radeon.modeset=1

5) Restartujemy komputer.

[edytuj] Późny start

1) Usuwamy wszystkie wpisy "vga=" i "video=" z /boot/grub/menu.lst (GRUB) lub /boot/grub/grub.cfg (GRUB2). Używanie innych buforów ramek, takich jak uvesafb/radeonfb, wywoła konflikt z KMS. Dlatego usuń wszystkie z MODULES w pliku /etc/mkinitcpio.conf.

2) Dodajemy "radeon" do MODULES w pliku /etc/mkinitcpio.conf.

3) Dodajemy "fbcon" do MODULES w pliku /etc/mkinitcpio.conf.

4) Niektórzy odnotowują przyspieszenie ładowania modułu udev po dodaniu "options radeon modeset=1" do /etc/modprobe.d/modprobe.conf.

5) Restartujemy komputer.

[edytuj] Wyłączenie KMS

1) Dodajemy "radeon.modeset=0" do pliku /boot/grub/menu.lst (GRUB) lub /boot/grub/grub.cfg (GRUB2).

Jeżeli wpis "radeon.modeset=0" nie będzie działał, można użyć zamiast niego wpisu "nomodeset".

2) Następnie wpisujemy w konsoli:

# mkinitcpio -p kernel26

3) Restartujemy komputer.

W niektórych przypadkach może okazać się przydatne stworzenie pliku /etc/modprobe.d/radeon.conf z zawartością: 

options radeon modeset=0

[edytuj] Słaba wydajność karty

Problem może być spowodowany domyślnie włączonym KMS. Warto to sprawdzić:

# dmesg

Jeżeli wynik będzie taki, jak poniżej, zalecane jest wyłączenie KMS.

# dmesg|egrep "drm|radeon"
 [drm] radeon default to kernel modesetting.
 ...
 [drm:radeon_driver_load_kms] *ERROR* Failed to initialize radeon, disabling IOCTL

Natomiast, jeżeli wynik będzie taki:

# dmesg|egrep "drm|radeon"
 platform radeon_cp.0: firmware: requesting radeon/R600_rlc.bin
 r600_cp: Failed to load firmware "radeon/R600_rlc.bin"
 [drm:r600_startup] *ERROR* Failed to load firmware!
 radeon 0000:01:00.0: disabling GPU acceleration

Zainstaluj linux-firmware.
