[edytuj] Wprowadzenie

GNOME Shell to tworzona od pewnego czasu alternatywa dla "normalnego" pulpitu GNOME. Jest to podstawa GNOME 3. Projekt zapewnia podstawowe funkcje interfejsu, takie jak przejście do okna i uruchamianie aplikacji, a także wykorzystuje możliwości nowoczesnych kart graficznych i wprowadza innowacyjne koncepcje interfejsu użytkownika, zapewniając przyjemne i łatwe użytkowanie środowiska.

Więcej informacji można znaleźć na stronie live.gnome.org (w języku angielskim).

Ostrzeżenie: GNOME Shell wymaga sprzętowej akceleracji grafiki. Upewnij się, że system graficzny działa przed próbą instalacji środowiska.

[edytuj] Instalacja

# pacman -Sy gnome-shell

Rozszerzenia powłoki z oficjalnego repozytorium:

# pacman -Sy gnome-shell-extensions

[edytuj] Przydatne informacje

Co potrafi GNOME-Shell?
Zajrzyj na stronę :)
Czy mogę w jakiś sposób rozszerzyć możliwości powłoki GNOME Shell?
Już przed premierą GNOME Shell społeczność linuksowa nie próżnowała, dzięki czemu w oficjalnym repozytorium można znaleźć pakiet gnome-shell-extensions zapewniający rozszerzenie możliwości tej powłoki. Zawiera on jednake wszystkie rozszerzenia. Jeżeli chesz tylko wybrane, poszukaj w repozytorium. Pamiętaj o ponownym uruchomieniu środowiska ([Alt]+[F2], nastepnie [r] i [Enter]). Ponadto z Arch User Repository można zainstalować gnome-shell-extensions-git.
Jeżeli chcesz dowiedzieć się więcej, przeczytaj artykuł ( w jęz. angielskim).
Ponadto w Arch User Repository znajdziesz inne rozszerzenia tworzone przez społeczność linuksową. Przykładem może byc pakiet gnome-shell-frippery, który zapewnia rozszerzenia:
- zmiana Podglądu na Menu znane z GNOME do wersji 2.32;
- skopiowanie ikon ulubionych aplikacji na panel;
- wyłączenie opcji dynamicznego pulpitu;
- przeniesienie zegara bliżej prawej strony panelu;
Więcej o powyższych rozszerzeniach dowiesz się na stronie ich twórcy.
Wartymi wspomnienia są również pakiety:
- gnome-shell-system-monitor-applet-git, który pokazuje na panelu zużycie pamięci RAM i obciążenie procesora.
- gnome-shell-extension-weather-git informujący na panelu o aktualnym stanie pogody.
- gnome-shell-extension-noa11y-git wyłączający widok ikony dostępności.
- gnome-shell-extension-mediaplayer-git, za pomocą którego można śledzicz poczynania odtwarzacza multimedialnego.
- gnome-shell-extension-activities-button, zmieniający Podgląd w logo Archa.
- gnome-shell-extension-autohide-topbar, pozwalający na schowanie górnego panelu.
- gnome-shell-extension-shellshape-git, który zorganizuje otwarte okna na pulpicie.
Rozszerzenie przestało działać po aktualizacji pakietu gnome-shell. Nie wiem co dalej...
Może mieć to związek ze zmianą wersji pakietu gnome-shell. W tej sytuacji możesz poczekać na aktualizację pakietu z rozszerzeniem, które przestało działać lub spróbować poniższego rozwiązania (może pomóc, ale nie musi):
Edytuj plik metadata.json w katalogu danego rozszerzenia i sprawdź, jakie wersje gnome-shell ono obsługuje:
File: metadata.json
{
	"shell-version": ["3.0.2"],
	"uuid": "noa11y.icon@panel.ui",
	"name": "noa11y",
	"description": "Turn off the a11y icon in the panel"
}
W przykładzie masz podaną wersję 3.0.2 - spróbuj zmienić na taką, którą masz zainstalowaną. Następnie zrestartuj powłokę ([Alt]+[F2]->[R]->[Enter]).
Nie podoba mi się GNOME-Shell. Chciałbym stary wygląd.
Jeżeli wolisz pulpit bez GNOME-Shell, wykorzystaj do tego tryb fallback. Włączyć/wyłączyć go możesz w ustawieniach systemowych (Informacje o systemie > Grafika > Wymuszenie trybu zastępczego) lub za pomocą polecenia:
$ GSETTINGS_BACKEND=dconf gsettings set org.gnome.desktop.session session-name gnome-fallback

Teraz wystarczy, że przelogujesz użytkownika.

Mam zasłabą kartę graficzną na GNOME-Shell. Co zrobić?
Przeczytaj powyższą informację.
Ja mogę usunąć z panelu ikonę dostępności?
W katalogu $HOME/.local/share/gnome-shell/extensions/noa11y.icon@panel.ui utwórz dwa pliki.
Pierwszy nazwany extension.js:
File: $HOME/.local/share/gnome-shell/extensions/noa11y.icon@panel.ui/extension.js
const Panel = imports.ui.panel;

function main() {
	Panel.STANDARD_TRAY_ICON_SHELL_IMPLEMENTATION['a11y'] = ;
}
Drugi nazwany metadata.json:
File: $HOME/.local/share/gnome-shell/extensions/noa11y.icon@panel.ui/metadata.json
{
 	"shell-version": ["3.0.1"],
	"uuid": "noa11y.icon@panel.ui",
	"name": "na11y",
 	"description": "Turn off the ally icon in the panel"
}
Na koniec zrestartuj środowisko lub system.
Ja mogę usunąć z panelu ikonę bluetooth?
W katalogu $HOME/.local/share/gnome-shell/extensions/nobluetooth.icon@panel.ui utwórz dwa pliki.
Pierwszy nazwany extension.js:
File: $HOME/.local/share/gnome-shell/extensions/nobluetooth.icon@panel.ui/extension.js
const Panel = imports.ui.panel;

function main() {
	Panel.STANDARD_TRAY_ICON_SHELL_IMPLEMENTATION['bluetooth'] = ;
}
Drugi nazwany metadata.json:
File: $HOME/.local/share/gnome-shell/extensions/nobluetooth.icon@panel.ui/metadata.json
{
	"shell-version": ["3.0.1"],
	"uuid": "nobluetooth.icon@panel.ui",
	"name": "nbluetooth",
	"description": "Turn off the bluetooth icon in the panel"
}
Na koniec zrestartuj środowisko lub system.
