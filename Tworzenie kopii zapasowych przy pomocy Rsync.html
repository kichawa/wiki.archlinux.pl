[edytuj] Wprowadzenie

Rsync to jedno z najlepszych narzędzi służących do wykonywania kopii zapasowych w świecie Linuksa. Kopiuje jedynie zmienione pliki i potrafi dokonywać kompresji danych "w locie". Potrafi także odzwierciedlać usuwanie plików. Doskonale nadaje się do synchronizacji drzew katalogów i plików, dzięki czemu możemy go używać do tworzenia na przykład kopii zapasowych serwisów WWW.

[edytuj] Podstawy tworzenia skryptu

Na początek musisz utworzyć dwa pliki. Pierwszym będzie skrypt, który utworzy kopię zapasową Twoich plików. Drugim jest plik, w którym tworzysz listę katalogów/plików nie uwzględnianych podczas tworzenia kopii oraz tych, które chcesz zachować, a znajdują się one w wykluczonych.

Poniżej przedstawiony jest przykład pierwszego skryptu - zachowane są dowiązania plików, prawa dostępu i inne atrybuty.

File: rbackup.sh
#!/bin/sh
# rsync backup script

sudo sh -c "
    rsync -av --delete-excluded --exclude-from=[plik_filrtujący] [/źródło/do/skopiowania] $1;
    touch $1[/ścieżka/do/tworzonej/kopii]
"

Na przykład:

File: rbackup.sh
#!/bin/sh
# rsync backup script

sudo sh -c "
    rsync -av --delete-excluded --exclude-from=backup.lst / $1;
    touch $1/media/backup
"
Źródło do skopiowania; /
W tym przypadku zostanie utworzona kopia całego drzewa root.
Ścieżka do utworzonej kopii; $1
Tu podajesz ścieżkę do miejsca, gdzie ma być zapisana kopia zapasowa.
Lista zachowanych/wykluczonych plików/katalogów; --exclude-from=backup.lst
Użyty w tym przykładzie plik backup.lst zawiera katalogi, których nie chcesz w swojej kopii oraz te, które chcesz zachować, a znajdują się one w wykluczonych. Taki plik może wyglądać następująco:
File: backup.lst
# Include
+ /dev/console
+ /dev/initctl
+ /dev/null
+ /dev/zero

# Exclude
- /dev/*
- /proc/*
- /sys/*
- /tmp/*
- *lost+found
- /media/backup/*

[edytuj] Tworzenie kopii zapasowej

  • Upewnij się, że miejsce, gdzie ma być zapisana kopia zapasowa, jest zamontowane.
  • Wpisz polecenie (wykorzystana tu jest nazwa skryptu opisanego powyżej):
# ./rbackup.sh /media/backup/

[edytuj] Boot setup

Po zakończeniu synchronizacji zostanie zmodyfikowany plik /etc/fstab w miejscu przeznaczenia kopii, gdzie również trzeba doinstalować bootloadera i skonfigurować poprawnie /boot/grub/menu.lst uwzględniając nowe lokacje.

[edytuj] Modyfikacja /etc/fstab

Edytuj plik /etc/fstab w kopii zapasowej:

File: /media/backup/etc/fstab
none         /dev/pts      devpts    defaults      0 0
none         /dev/shm      tmpfs     defaults      0 0

/dev/sda1    /boot         ext4      defaults      0 1
/dev/sda5    /var          ext4      defaults      0 1
/dev/sda6    /usr          ext4      defaults      0 1
/dev/sda7    /             ext4      defaults      0 1
/dev/sda8    /home         ext4      defaults      0 1
/dev/sda9    swap          swap      defaults      0 0

Ponieważ Rsync tworzy kopię całego systemu plików, należy w pliku fstab znajdującym się w miejscu docelowym kopii zmienić zawartość na przykładzie:

File: /media/backup/etc/fstab
none         /dev/pts      devpts    defaults      0 0
none         /dev/shm      tmpfs     defaults      0 0

/dev/sdb1    /             ext4      defaults      0 1
Ostrzeżenie: To jest tylko przykład. U Ciebie wpis może się różnić.

[edytuj] Instalacja bootloadera

Wpisz w konsoli:

# grub

Następnie:

root (hd1,0)
setup (hd1)

Wyjaśnienie:

root; hd 1,0
Lokalizacja plików GRUB-a, w tym przypadku "hd 1" oznacza drugie urządzenie (/dev/sdb), natomiast "0" pierwszą partycję (/dev/sdb1).
setup; hd 1
Lokalizacja zainstalowanego bootloadera, w tym przypadku znajduje się on w MBR na drugim urządzeniu.

[edytuj] Konfiguracja bootloadera

Problem polega na tym, że nawet jeśli bootloader instaluje się poprawnie, jego pozycje menu są do głównego systemu partycji, a nie systemu kopii zapasowyej. Problem można rozwiązać przez utworzenie niestandardowego pliku /boot/grub/menu.lst dla kopii. W tym celu wykorzystaj rbackup.sh:

File: rbackup.sh
#!/bin/sh
# rsync backup script

sudo sh -c "
    rsync -av --delete-excluded --exclude-from=backup.lst / $1;
    cp ~/custom.menu.lst $1/boot/grub/menu.lst;
    touch $1/BACKUP
"

[edytuj] Przykłady skryptów

[edytuj] Codzienna kopia zapasowa

Jeżeli masz zainstalowany pakiet cron, umieść skrypt w na przykład /etc/cron.daily.

File: /etc/cron.daily/backup
#!/bin/bash
rsync -a --delete /źródło/do/skopiowania /ścieżka/do/tworzonej/kopii &> /dev/null

Opcja -a oznacza zachowanie wszystkich atrybutów archiwizowanych plików.

Opcja --delete oznacza usunięcie tych plików z kopii, których nie ma już w źródle.

[edytuj] Tygodniowa kopia różnicowa

File: /etc/cron.daily/backup
#!/bin/bash

DAY=$(date +%A)

if [ -e /location/to/backup/incr/$DAY ] ; then
  rm -fr /location/to/backup/incr/$DAY
fi

rsync -a --delete --inplace --backup --backup-dir=/location/to/backup/incr/$DAY /folder/to/backup/ /location/to/backup/full/ &> /dev/null
