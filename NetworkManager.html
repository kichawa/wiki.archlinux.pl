[edytuj] Wprowadzenie

NetworkManager jest menedżerem połączeń sieciowych, zarządza interfejsami sieci przewodowych i bezprzewodowych. Jest domyślnie wykorzystywany w GNOME.

[edytuj] Instalacja

  • GNOME / XFCE
# pacman -S networkmanager network-manager-applet

Jeżeli chcesz posiadać w GNOME możliwość zapisu uwierzytelnienia, zainstaluj pakiet:

# pacman -S gnome-keyring
  • KDE
# pacman -S networkmanager kdeplasma-applets-networkmanagement
  • Openbox
# pacman -S network-manager-applet xfce4-notifyd hicolor-icon-theme gnome-icon-theme

Jeżeli chcesz posiadać możliwość zapisu uwierzytelnienia, zainstaluj pakiet:

# pacman -S gnome-keyring
  • Zalecana jest instalacja D-Bus:
# pacman -S dbus

[edytuj] Konfiguracja

  • Upewnij się za pomocą polecenia hostname, czy masz wpisaną nazwę hosta. Jest ona unikalna i utworzona w celu identyfikacji urządzenia w sieci.

W razie problemów, nazwę hosta ustalasz w:

File: /etc/rc.conf
HOSTNAME="nazwa_hosta"
File: /etc/hosts
127.0.0.1      nazwa_hosta.domain.org localhost.localdomain      localhost nazwa_hosta

Aby tymczasowo ustawić nazwę hosta, wystarczy wpisać:

# hostname nazwa_hosta
  • Następnie w pliku /etc/rc.conf w sekcji INTERFACES wyłącz uruchamianie wszystkich interfejsów sieciowych:
File: /etc/rc.conf
# -----------------------------------------------------------------------
# NETWORKING
# -----------------------------------------------------------------------
#
# HOSTNAME: Hostname of machine. Should also be put in /etc/hosts
#
HOSTNAME="nazwa_hosta"

...

# interface=eth0
# address=
# netmask=
# gateway=

...
  • W sekcji DEAMONS wyłącz wpis network i dopisz networkmanager i dbus:
File: /etc/rc.conf
DAEMONS=(...!network dbus networkmanager...)
  • Trzeba także dodać użytkownika do wymaganej grupy:
# gpasswd -a nazwa_użytkownika network

W razie problemów utwórz grupę networkmanager i dodaj do niej siebie:

# groupadd networkmanager
# gpasswd -a nazwa_użytkownika networkmanager

[edytuj] Konfiguracja w Openboksie

Aby zapobiec błędom D-Bus, w pliku ~/.xinitrc polecenie uruchamiające Openboksa powinno wyglądać następująco:

File: ~/.xinitrc
exec ck-launch-session openbox-session

Zdarza się u niektórych użytkowników, że NetworkManager nie uruchamia się prawidłowo. Aby to naprawić, należy usunąć plik /etc/xdg/autostart/nm-applet.desktop (także po aktualizacji pakietu) i dodać do ~/.config/Openbox/autostart.sh:

File: ~/.config/Openbox/autostart.sh
(sleep 3 && /usr/bin/nm-applet --sm-disable) &
