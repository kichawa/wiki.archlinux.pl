[edytuj] Wprowadzenie

Podczas uruchamiania systemu operacyjnego na ekranie pojawia się dużo nie zawsze zrozumiałego tekstu. Jest to proces nazywany sekwencją startową. Najpierw bootloader ładuje obraz jądra systemu do pamięci i zleca procesorowi jego wykonanie. W chwili, kiedy jądro zostanie załadowane i wykonane, uruchamiane są zadania związane ściśle z jądrem, po czym uruchamiany jest proces init. Proces ten następnie upewnia się czy wszystkie systemy plików (zdefiniowane w /etc/fstab) zostały poprawnie zamontowane i są gotowe do pracy. Następnie uruchamiane są pozostałe usługi z katalogu /etc/init.d, niezbędne do poprawnego działania systemu. Na koniec, kiedy wszystkie skrypty zostaną wykonane, init aktywuje terminale przy pomocy służącego do tego programu pod nazwą agetty.

Oczywiście o wszystkim nie decyduje sam init. Potrzebuje on stosownego pliku konfiguracyjnego, który zawiera informacje o zadaniach jakie ma wykonać. Ten plik to /etc/inittab.

Możesz zorientować się, gdzie leży init w hierarchii procesu systemu, wykonując polecenie podane w poniższym przykładzie:

$ pstree -caA
init  
  |-acpid
  |-crond -S -l info
  |-dbus-daemon --system
  |-slim
  |   |-X -nolisten tcp vt07 -auth /var/run/slim.auth
  |   `-ratpoison
  |       |-terminal -x sh -c TERM=xterm-256color; exec tmux -2 -l -u -v
  |       |   |-gnome-pty-helpe
  |       |   |-tmux -2 -l -u -v
  |       |   `-{terminal}
  |       `-xscreensaver -no-splash
  |-sshd
  |-synergyc --daemon --name askapache --restart askapache.firewire:64800
  |   |-{synergyc}
  |   `-{synergyc}
  |-syslog-ng
  |   `-syslog-ng
  |-tmux -2 -l -u -v
  |   `-bash
  |       `-pstree -caA
  `-udevd --daemon
      `-udevd --daemon

Poniżej masz podane główne skrypty startowe:

  • rc::sysinit:/etc/rc.sysinit
  • rs:S1:wait:/etc/rc.single
  • rm:2345:wait:/etc/rc.multi
  • rh:06:wait:/etc/rc.shutdown

[edytuj] Automatyczne logowanie w konsoli

[edytuj] agetty

Opiszę kilka wariantów i podam czym się różnią..

[edytuj] Z hasłem, bez pytania o użytkownika

File: /usr/sbin/agetty_autologin_user.sh
#!/bin/bash - 

exec login -- nazwa_uzytkownika

Teraz ustaw plikowi chmod na 700 i ustaw takie coś w pliku /etc/inittab: 

c6:2345:respawn:/sbin/agetty -l /usr/sbin/agetty_autologin_user.sh -n -8 38400 tty6 linux

od teraz na konsoli 6 prosi tylko o podanie hasła dla użytkownika podanego w skrypcie.

[edytuj] Bez hasła, bez pytania o użytkownika

File: /usr/sbin/agetty_autologin_user.sh
#!/bin/bash - 

exec login -f nazwa_uzytkownika

Teraz ustaw plikowi chmod na 700 i ustaw takie coś w pliku /etc/inittab: 

c6:2345:respawn:/sbin/agetty -l /usr/sbin/agetty_autologin_user.sh -n -8 38400 tty6 linux

od teraz na konsoli 6 jest automatyczne logowanie na podanego użytkownika.

[edytuj] Bez hasła, z pytaniem o użytkownika

File: /usr/sbin/agetty_autologin_user.sh
#!/bin/bash - 

exec login -f "${2}"

Teraz ustaw plikowi chmod na 700 i ustaw takie coś w pliku /etc/inittab: 

c6:2345:respawn:/sbin/agetty -l /usr/sbin/agetty_autologin_user.sh -8 38400 tty6 linux

od teraz na konsoli 6 prosi tylko o nazwę użytkownika.

Ostrzeżenie: W skrypcie zalecam skorzystać z instrukcji warunkowych case lub if i ograniczyć do określonych użytkowników, inaczej np. root też zostanie zalogowany bez hasła. Przykład:
File: /usr/sbin/agetty_autologin_user.sh
#!/bin/bash - 

case "${2}" in
  "romek")  exec login -f romek;;
  "atomek") exec login -f atomek;;
  *)        exec login -- "${2}";;
esac

[edytuj] Co tu zostało zrobione?

  • Jeżeli skrypt bash uruchamia program login z opcją -f, wtedy jest pomijane hasło
  • agetty (opcja -l) - wykonuj niestandardowy program logowania (wspomniany skrypt bash)
  • agetty (opcja -n) - nie pytaj o nazwę użytkownika..

[edytuj] mingetty

Zainstaluj pakiet mingetty z Arch User Repository. Następnie w pliku /etc/inittab zmień:

File: /etc/inittab
c1:2345:respawn:/sbin/agetty -8 38400 tty1 linux
c2:2345:respawn:/sbin/agetty -8 38400 tty2 linux
c3:2345:respawn:/sbin/agetty -8 38400 tty3 linux
c4:2345:respawn:/sbin/agetty -8 38400 tty4 linux
c5:2345:respawn:/sbin/agetty -8 38400 tty5 linux
c6:2345:respawn:/sbin/agetty -8 38400 tty6 linux

Na:

File: /etc/inittab
c1:2345:respawn:/sbin/mingetty --autologin nazwa_użytkownika tty1 linux
c2:2345:respawn:/sbin/agetty -8 38400 tty2 linux
c3:2345:respawn:/sbin/agetty -8 38400 tty3 linux
c4:2345:respawn:/sbin/agetty -8 38400 tty4 linux
c5:2345:respawn:/sbin/agetty -8 38400 tty5 linux
c6:2345:respawn:/sbin/agetty -8 38400 tty6 linux

[edytuj] Uruchomienie sewera X

Ostrzeżenie: Metoda ta nie będzie używać /bin/login lub rejestrować sesję. Twoja sesja nie będzie dopuszczona jako lokalna przez consolekit, więc nie będzie można zamknąć czy zrestartować systemu bez konieczności użycia su lub sudo.

Większość użytkowników, którzy zamierzają uruchomić serwer X podczas uruchamiania, będzie chciała zainstalować menedżer logowania. Alternatywą dla niego może być edycja /etc/inittab. Wystarczy zamienić:

File: /etc/inittab
id:3:initdefault:
[...]
x:5:respawn:/usr/bin/xdm -nodaemon

Na:

File: /etc/inittab
id:5:initdefault:
[...]
x:5:once:/bin/su -l -c '/usr/bin/startx </dev/null' nazwa użytkownika

Ostatni wiersz użyty jest do uruchamiania serwera X wykorzystując /etc/profile.
