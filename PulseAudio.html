[edytuj] Wprowadzenie

PulseAudio to serwer dźwięku dla Linuksa. Jego zaletami są między innymi: możliwość regulacji głośności dla każdej aplikacji (korzystającej z serwera) z osobna, lepsze wsparcie dla urządzeń audio-usb (hotplug), a także możliwość łączenia urządzeń audio. Jest domyślnie wykorzystany w GNOME.

[edytuj] Instalacja

Pakiet znajduje się w [Community], zatem wystarczy wpisać:

# pacman -S pulseaudio

Aby umożliwić sobie konfigurację z poziomu okienek należy doinstalować:

  • z oficjalnego repozytorium
# pacman -S paprefs pavucontrol
  • z Arch User Repository (na przykład przy pomocy Yaourt)
$ yaourt -S pavumeter

[edytuj] Uruchomienie

Upewnij się, że D-Bus jest uruchomiony. Następnie wpisz:

$ pulseaudio --start

Jeżeli "posiadasz pulpit graficzny", PulseAudio uruchomisz za pomocą polecenia:

$ start-pulseaudio-x11

W przypadku środowisk KDE lub GNOME jest bardzo prawdopodobne, że PulseAudio uruchomi się automatycznie wraz ze startem systemu.

[edytuj] Kompatybilność z ALSA

Ważne jest też zachowanie kompatybilności z aplikacjami, korzystającymi jedynie z ALSY:

# pacman -S pulseaudio-alsa

Następnie edytujemy plik /etc/asound.conf i upewniamy się, że poniższe wiersze wyglądają tak:

File: /etc/asound.conf
pcm.pulse {
    type pulse
}
ctl.pulse {
    type pulse
}
pcm.!default {
    type pulse
}
ctl.!default {
    type pulse
}

Dwa ostatnie wiersze odpowiadają za to, że PulseAudio będzie używany jako domyślny.

Dodatkowo w pliku /etc/rc.conf wyłączamy lub usuwamy moduł snd-pcm-oss.

[edytuj] Kompatybilność z OSS

Aby zapobiec temu, że aplikacja korzystająca z emulacji ALSA OSS omija PulseAudio, zaleca się usunięcie modułu snd-pcm-oss:

# rmmod snd-pcm-oss

Aplikacje korzystające z OSS uruchamiamy przykładowo:

$ padsp nazwa_programu

[edytuj] GStreamer

Wpisz w konsoli:

$ gstreamer-properties

i wybierz serwer dźwięku PulseAudio, zarówno wejście, jak i wyjście audio.

[edytuj] PulseAudio i aplikacje

  • Audacious wspiera PulseAudio. Wystarczy w ustawieniach dźwięku wybrać PulseAudio Output Plugin.
  • Aby MPlayer korzystał z tego serwera dźwięku, należy w jego pliku konfiguracyjnym dodać/zmienić poniższy wiersz:
File: ~/.mplayer/config
ao=pulse
  • W razie problemów z dźwiękiem w flashu, zainstaluj libflashsupport-pulse. Ponadto, jeżeli posiadasz zainstalowanego Arch x86_64 i używasz flashplayera z repozytorium [multilib], doinstaluj pakiety lib32-alsa-plugins oraz lib32-libcanberra-pulse.

[edytuj] Equalizer

Linux pozwala ulepszyć sobie dźwięk według własnych wymogów. Zainstaluj pakiet pulseaudio-equalizer. Narzędzie możesz skonfigurować na dwa sposoby:

  • z poziomu konsoli - aby dowiedzieć się więcej, wpisz:
$ pulseaudio-equalizer
  • za pomocą graficznego konfiguratora - wpisz:
$ pulseaudio-equalizer-gtk
Notka: Z Arch User Repository można również zainstalować pulseaudio-equalizer-bzr.
