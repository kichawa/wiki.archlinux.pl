[edytuj] Wprowadzenie

Środowisko graficzne GNOME (ang. GNU Network Object Model Environment) jest częścią projektu GNU i zostało wydane na licencji GPL. Projekt został rozpoczęty w sierpniu 1997 roku przez Miguela de Icaza w celu stworzenia w pełni wolnodostępnego graficznego środowiska użytkownika dla systemu GNU/Linux, jako alternatywy dla środowiska KDE, opartego o bibliotekę Qt (do listopada 1997 nie będącą wolnym oprogramowaniem). Środowisko GNOME składa się wyłącznie z wolnego i otwartego oprogramowania.

Projekt GNOME 3 ma na celu stworzenie od podstaw całkowicie nowego i nowoczesnego pulpitu.

Obecna, najnowsza wersja środowiska GNOME zapewnia dwa tryby:

  • GNOME Shell - alternatywa dla "normalnego" pulpitu GNOME;
  • Tryb zastępczy - pulpit przypominający ten znany z wersji 2.32 i starszej;

Domyślnym pulpitem jest ten pierwszy. Uzyskujemy tu ciekawe efekty kosztem zwiększenia wymagań sprzętowych.

Tryb zstępczy uruchamiany jest, gdy system uzna, iż nasz sprzęt nie jest wystarczającuy dla GNOME Shell.

Uwaga!
Na chwilę obecną sterowniki ATI Catalyst nie współgrają prawidłowo z GNOME Shell.

[edytuj] Instalacja

Upewnij się, że masz zainstalowaną grupę xorg oraz jej zależności:

# pacman -Sy --needed xorg

Teraz wystarczy, że wykonasz polecenie:

# pacman -S gnome

Dodatkowe pakiety, zwiększające funkcjonalność środowiska, znajdują się w grupie gnome-extra:

# pacman -S gnome-extra

Pacman zapewne spyta się, które pakiety z grupy zainstalować (każdemu przypisuje numer). Jeżeli wiesz, które chesz, wpisz ich numery oddzielając je spacją. Jeżeli zdecydujesz sie wszystkie pakiety z grupy, wciśnij [Enter].

[edytuj] Uruchomienie GNOME

GNOME uruchomimy z konsoli za pomocą polecenia:

$ xinit gnome-session

Aby sesja była zapamiętana, dodaj poniższy wpis do pliku ~ /.xinitrc (upewnij się, że jest to jedyna linia, która zaczyna się od exec):

File: ~/.xinitrc
exec dbus-launch ck-launch-session gnome-session

[edytuj] GNOME Display Manager

GDM (GNOME Display Manager) to menedżer logowania dla środowiska graficznego GNOME. Został zaprojektowany jako prostsza w konfiguracji alternatywa dla domyślnego w X11 menedżera XDM.

# pacman -S gdm

a następnie na końcu sekcji DAEMONS w pliku /etc/rc.conf dodajemy odpowiedni wpis:

File: /etc/rc.conf
DAEMONS=(...gdm)

Jeżeli chcesz przeczytać więcej GDM, odwiedź artykuł.

[edytuj] Zalecane demony i moduły

  • D-Bus

Na początek instalujemy demona:

# pacman -S dbus

Następnie edytujemy plik /etc/rc.conf:

File: /etc/rc.conf
DAEMONS = (...dbus...)

Na koniec ręcznie uruchomiamy DBUS wpisując w linii poleceń:

# /etc/rc.d/dbus start

Aby w pełni wykorzystać możliwości DBUS w swoim systemie, użytkownik musi należeć do wymaganych grup, a zatem:

# gpasswd -a nazwa_użytkownika optical # gpasswd -a nazwa_użytkownika storage # gpasswd -a nazwa_użytkownika dbus # gpasswd -a nazwa_użytkownika disk

Jeżeli włączyliśmy DBUS przed dopisaniem użytkownika do grup, należy zrestartować demona przy pomocy poleceń:

# /etc/rc.d/dbus restart
  • GVFS

Aby korzystać z możliwości montowania wirtualnych systemów plików, zainstaluj oraz dodaj moduł fuse do pliku /etc/rc.conf:

File: /etc/rc.conf
MODULES= (...fuse...)

[edytuj] Konfiguracja

W tym miejscu dowiesz się, jak wiele istnieje metod, aby skonfigurować GNOME 3.

[edytuj] Konfiguracja za pomocą dconf-editor

Wskazówka: Ustawienia, których brakuje w dconf-editor, znajdziesz w gconf-editor.

GNOME 3 skonfigurować możesz za pomocą przejrzystego narzędzia dconf-editor. Poniżej masz opisane, w której zakładce znajdziesz przykładowe opcje:

org.gnome.desktop 
org.gnome.desktop.interface.font-name                         # Wybór czcionki systemowej
org.gnome.desktop.interface.cursor-theme                      # Wybór motywu kursora
org.gnome.desktop.interface.icon-theme                        # Wybór motywu ikon
org.gnome.desktop.interface.gtk-theme                         # Wybór motywu GTK
org.gnome.desktop.interface.buttons-have-icons                # Włączanie widoku ikon w przyciskach
org.gnome.desktop.interface.menus-have-icons                  # Włączanie widoku ikon w menu
org.gnome.desktop.background.picture-filename                 # Ustawienie ścieżki do tapety
org.gnome.settings-daemon 
org.gnome.settings-daemon.plugins.xsettings                   # Konfiguracja czcionki
org.gnome.shell 
org.gnome.shell.calendar                                      # Konfiguracja kalendarza
org.gnome.shell.clock                                         # Konfiguracja zegara

[edytuj] Konfiguracja z poziomu konsoli za pomocą gsettings

W GNOME 3 gsettings zastąpiło gconftool-2 stosowane w starszej wersji tego środowiska. Aby dowiedzieć się więcej, wpisz:

$ gsettings

Przykład oparty na konfiguracji zegara:

$ gsettings set org.gnome.shell.clock show-date true
$ gsettings set org.gnome.shell.clock show-seconds true

Polecenia aktywują odpowiednio wyświetlanie daty i upływających sekund.

Kolejnym przykładem jest poniższe polecenie aktywujące środkowy przycisk myszy:

$ gsettings set org.gnome.settings-daemon.peripherals.mouse middle-button-enabled true

Przykład wyszukiwania ustawień GNOME-Shell:

$ $ gsettings list-schemas | grep shell
org.gnome.shell.clock
org.gnome.shell.calendar
org.gnome.shell
org.gnome.shell.recorder

[edytuj] Konfiguracja za pomocą pliku settings.ini

W pliku ~/.config/gtk-3.0/settings.ini istnieje mozliwość konfiguracji niektórych elementów (podobnie jak w ~/.gtkrc-2.0 dla GTK2+), na przykład: 

[Settings]
gtk-theme-name="Adwaita"
gtk-icon-theme-name="MeliaeSVGArch"
gtk-font-name="Ubuntu 10"
gtk-cursor-theme-name="Adwaita"
gtk-cursor-theme-size=24
gtk-toolbar-style=GTK_TOOLBAR_TEXT
gtk-toolbar-icon-size=GTK_ICON_SIZE_SMALL_TOOLBAR
gtk-button-images=0
gtk-menu-images=0
gtk-enable-event-sounds=0
gtk-enable-input-feedback-sounds=0

Przykład zawiera m.in. konfigurację motywu GTK, ikon oraz kursora, ustawienia wyglądu przycisków i menu oraz odtwarzania dźwięku.

Jeśli ta metoda nie działa, spróbuj dowiązania symbolicznego do tematu Adwaita:

# ln -s /usr/share/themes/Adwaita/gtk-3.0 ~/.config/gtk-3.0

[edytuj] Konfiguracja za pomocą narzędzia GNOME Tweak Tool

Zainstaluj narzędzie gnome-tweak-tool i uruchom. Dostaniesz dzięki temu dostęp do wielu ustawień systemu, na przykład wyboru widoku motywów ikon, Metacity i GTK czy możliwości włączenia/wyłączenia ikon w menu i w przyciskach. Ponadto GNOME Tweak Tool umożliwia włączenie przycisku minimalizacji czy maksymalizacji, konfigurację czcionki, a także ustawienie zachowania systemu po zamknięciu pokrywy laptopa.

Ponadto narzędzie umożliwia zarządzanie rozszerzeniami GNOME Shell.

[edytuj] Konfiguracja Nautilusa

Rozszerzenia:

  • nautilus-actions
  • nautilus-open-terminal

Aby móc w Nautilusie wpisywać ręcznie lokację, wciśnij [Ctrl] + [l]. Aby ustawić tą opcję na stałe, wpisz w konsoli:

$ gsettings set org.gnome.nautilus.preferences always-use-location-entry true

[edytuj] Ustawienia Metacity

Jeżeli nie zmieniłeś w trybie zastępczym domyślnego menedżera okien, używasz w tym momencie Metacity. Warto więc aby trochę tu pozmieniać, tak aby nie było zarysów okien podczas ich minimalizacji czy też szachownicy w trakcie przesuwania okna.

W tym celu gconf-editor i włącz następujące opcje: 

/ > apps > metacity > general > reduced_resources
/ > apps > metacity > general > compositing_manager
/ > desktop > gnome > interface > accessibility

Jeżeli chcesz zmniejszyć wysokość paska tytułowego, użyj polecenia:

# sed -i '/title_vertical_pad/s|value="[0-9]\{1,2\}"|value="0"|g' /usr/share/themes/Adwaita/metacity-1/metacity-theme-3.xml

GNOME pozwala na więcej - poniższe polecenie ukrywa belkę tytułową podczas maksymalizacji okna:

# sed -i -r 's|(<frame_geometry name="max")|\1 has_title="false"|' /usr/share/themes/Adwaita/metacity-1/metacity-theme-3.xml

Aby zapobiec nadpisaniu pliku metacity-theme-3.xml, dodaj odpowiedni wpis do /etc/pacman.conf:

File: /etc/pacman.conf
...
NoUpgrade = usr/share/themes/Adwaita/metacity-1/metacity-theme-3.xml
...

[edytuj] FAQ

Czy mój komputer będzie mógł uruchomić GNOME 3?
Środowisko GNOME 3 wymaga sprzętowego przyspieszenia grafiki, aby dostarczyć nowoczesny pulpit i pełnię możliwości. Nie ma się jednak co martwić w przypadku starszego sprzętu: GNOME 3 dostarczy interfejs zapasowy tzw. "Tryb Fallback", który zapewni użytkowanie w przypadku braku przyspieszenia sprzętowego.
Czy będę mógł używać swoich ulubionych programów GNOME w GNOME 3?
Tak. Istniejące programy środowiska GNOME będą działały doskonale w GNOME 3.
Co się stanie z poprzednią wersją środowiska GNOME?
Środowisko GNOME 2 miało długi żywot, i pewne jego części stały się trudne do utrzymania przez ten czas. W wyniku tego kontynuowanie wydawania całego środowiska GNOME 2 stało się niepraktyczne i pewne części poprzedniego pulpitu GNOME 2 nie otrzymają nowych wydań po wydaniu GNOME 3. Tradycyjny pulpit GNOME 2 jednak nie zniknie, będzie on wspierany przez dystrybucje jeszcze przez wiele lat.
Dlaczego pulpit GNOME 3 zaprojektowano właśnie w ten sposób?
Wiele pracy włożono w projektowanie środowiska GNOME 3, a każda decyzja projektowa została poważnie przemyślana. Odpowiedzi na pytania o konkretne decyzje można znaleźć na stronie FAQ środowiska GNOME 3.


[edytuj] Ogólne

Jak można usunąć niepotrzebne aktywatory z zakładki "Programy"?
W GNOME-Shell jest to niemożliwe. Musisz przejść do trybu zstępczego, następnie prawoklik na Menu - dopieru tu możesz włączyć/wyłączyć widoczność aktywatorów. Jeżeli chcesz edytować aktywatory, znajdują sie w katalogu /usr/share/applications. Możesz je edytować jedynie jako root. :
Ostrzeżenie: Edycja aktywatorów nie jest zalecana. Pamiętaj o zrobieniu kopii bezpieczeństwa przed edycją plików.
Warto wspomnieć o znanym już edytorze Alacarte. Obecnie nie jest on już używany domyślnie w GNOME - można za to zainstalować go z Arch User Repository.
Jak mogę ustawić, aby dany program uruchamiał się w raz z systemem?
Wykorzystaj do tego celu:
$ gnome-session-properties
W jaki sposób można wyłączyć/zrestartować system?
Na chwilę obecną opcja wyłączenia systemu z menu statusu jest ukryta. Aby zakończyć pracę z Archem, naciśnij i przytrzymaj Alt - w tym momencie w menu zamiast opcji uśpienia powinna widnieć opcja wyłączająca/restartująca system.
Ewentualnie wykorzystaj rozszerzenie "Alternative Status Menu" z pakietu gnome-shell-extensions - będziesz miał wtedy w Menu na stałe widoczną opcję wyłączającą system.
Brakuje ikony baterii na panelu. Dlaczego?
Sprawdź, czy nie brakuje pakietu:
$ pacman -Qs gnome-power-manager
Czy Xmonad współgra z GNOME 3?
Działa, ale tylko w trybie zastepczym. Musisz posiadać dwa poniższe pliki:
File: /usr/share/gnome-session/sessions/xmonad.session
[GNOME Session]
Name=Xmonad session
RequiredComponents=gnome-panel;gnome-settings-daemon;
RequiredProviders=windowmanager;notifications;
DefaultProvider-windowmanager=xmonad
DefaultProvider-notifications=notification-daemon
File: {{{name}}}
[Desktop Entry]
Name=Xmonad GNOME
Comment=Tiling window manager
TryExec=/usr/bin/gnome-session
Exec=gnome-session --session=xmonad
Type=XSession

Teraz wystarczy, że przelogujesz się i podczas wyboru sesji wybierzesz Xmonad GNOME.


[edytuj] GNOME Shell

Co potrafi GNOME-Shell?
Zajrzyj na stronę :)
Czy mogę w jakiś sposób rozszerzyć możliwości powłoki GNOME Shell?
Już przed premierą GNOME Shell społeczność linuksowa nie próżnowała, dzięki czemu w oficjalnym repozytorium można znaleźć pakiet gnome-shell-extensions zapewniający rozszerzenie możliwości tej powłoki. Zawiera on jednake wszystkie rozszerzenia. Jeżeli chesz tylko wybrane, poszukaj w repozytorium. Pamiętaj o ponownym uruchomieniu środowiska ([Alt]+[F2], nastepnie [r] i [Enter]). Ponadto z Arch User Repository można zainstalować gnome-shell-extensions-git.
Jeżeli chcesz dowiedzieć się więcej, przeczytaj artykuł ( w jęz. angielskim).
Ponadto w Arch User Repository znajdziesz inne rozszerzenia tworzone przez społeczność linuksową. Przykładem może byc pakiet gnome-shell-frippery, który zapewnia rozszerzenia:
- zmiana Podglądu na Menu znane z GNOME do wersji 2.32;
- skopiowanie ikon ulubionych aplikacji na panel;
- wyłączenie opcji dynamicznego pulpitu;
- przeniesienie zegara bliżej prawej strony panelu;
Więcej o powyższych rozszerzeniach dowiesz się na stronie ich twórcy.
Wartymi wspomnienia są również pakiety:
- gnome-shell-system-monitor-applet-git, który pokazuje na panelu zużycie pamięci RAM i obciążenie procesora.
- gnome-shell-extension-weather-git informujący na panelu o aktualnym stanie pogody.
- gnome-shell-extension-noa11y-git wyłączający widok ikony dostępności.
- gnome-shell-extension-mediaplayer-git, za pomocą którego można śledzicz poczynania odtwarzacza multimedialnego.
- gnome-shell-extension-activities-button, zmieniający Podgląd w logo Archa.
- gnome-shell-extension-autohide-topbar, pozwalający na schowanie górnego panelu.
- gnome-shell-extension-shellshape-git, który zorganizuje otwarte okna na pulpicie.
Rozszerzenie przestało działać po aktualizacji pakietu gnome-shell. Nie wiem co dalej...
Może mieć to związek ze zmianą wersji pakietu gnome-shell. W tej sytuacji możesz poczekać na aktualizację pakietu z rozszerzeniem, które przestało działać lub spróbować poniższego rozwiązania (może pomóc, ale nie musi):
Edytuj plik metadata.json w katalogu danego rozszerzenia i sprawdź, jakie wersje gnome-shell ono obsługuje:
File: metadata.json
{
	"shell-version": ["3.0.2"],
	"uuid": "noa11y.icon@panel.ui",
	"name": "noa11y",
	"description": "Turn off the a11y icon in the panel"
}
W przykładzie masz podaną wersję 3.0.2 - spróbuj zmienić na taką, którą masz zainstalowaną. Następnie zrestartuj powłokę ([Alt]+[F2]->[R]->[Enter]).
Czy mogę za pomocą jakiegoś narzędzia zarządzać rozszerzeniami?
Tak, zainstaluj pakiet gnome-tweak-tool.
Nie podoba mi się GNOME-Shell. Chciałbym stary wygląd.
Jeżeli wolisz pulpit bez GNOME-Shell, wykorzystaj do tego tryb fallback. Włączyć/wyłączyć go możesz w ustawieniach systemowych (Informacje o systemie > Grafika > Wymuszenie trybu zastępczego) lub za pomocą polecenia:
$ GSETTINGS_BACKEND=dconf gsettings set org.gnome.desktop.session session-name gnome-fallback

Teraz wystarczy, że przelogujesz użytkownika.

Mam zasłabą kartę graficzną na GNOME-Shell. Co zrobić?
Przeczytaj powyższą informację.
Ja mogę usunąć z panelu ikonę dostępności?
W katalogu $HOME/.local/share/gnome-shell/extensions/noa11y.icon@panel.ui utwórz dwa pliki.
Pierwszy nazwany extension.js:
File: $HOME/.local/share/gnome-shell/extensions/noa11y.icon@panel.ui/extension.js
const Panel = imports.ui.panel;

function main() {
	Panel.STANDARD_TRAY_ICON_SHELL_IMPLEMENTATION['a11y'] = ;
}
Drugi nazwany metadata.json:
File: $HOME/.local/share/gnome-shell/extensions/noa11y.icon@panel.ui/metadata.json
{
 	"shell-version": ["3.0.1"],
	"uuid": "noa11y.icon@panel.ui",
	"name": "na11y",
 	"description": "Turn off the ally icon in the panel"
}
Na koniec zrestartuj środowisko lub system.
Jeżeli ta metoda nie pomoże, w pliku /usr/share/gnome-shell/js/ui/panel.js zakomentuj poniższą linię
File: /usr/share/gnome-shell/js/ui/panel.js
'a11y': imports.ui.status.accessibility.ATIndicator,
aby wyglądała tak:
File: /usr/share/gnome-shell/js/ui/panel.js
/* 'a11y': imports.ui.status.accessibility.ATIndicator, */
Ja mogę usunąć z panelu ikonę bluetooth?
W katalogu $HOME/.local/share/gnome-shell/extensions/nobluetooth.icon@panel.ui utwórz dwa pliki.
Pierwszy nazwany extension.js:
File: $HOME/.local/share/gnome-shell/extensions/nobluetooth.icon@panel.ui/extension.js
const Panel = imports.ui.panel;

function main() {
	Panel.STANDARD_TRAY_ICON_SHELL_IMPLEMENTATION['bluetooth'] = ;
}
Drugi nazwany metadata.json:
File: $HOME/.local/share/gnome-shell/extensions/nobluetooth.icon@panel.ui/metadata.json
{
	"shell-version": ["3.0.1"],
	"uuid": "nobluetooth.icon@panel.ui",
	"name": "nbluetooth",
	"description": "Turn off the bluetooth icon in the panel"
}
Na koniec zrestartuj środowisko lub system.
