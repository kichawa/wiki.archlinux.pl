[edytuj] Wprowadzenie

ALSA, to system obsługi dźwięku, domyślnie zaimplementowany w jądrze Linuksa. Obecnie wszystkie niezbędne moduły powinny zostać wykryte i załadowane automatycznie przez mechanizm Udev.

[edytuj] Instalacja

  • Wymagane pakiety przez programy korzystające z ALSY:
# pacman -S alsa-lib alsa-utils

Opcjonalnie:

# pacman -S alsa-firmware
  • Zalecane pakiety, jeżeli chcesz używać aplikacji ze wsparciem dla OSS:
# pacman -S alsa-oss
  • Dodaj odpowiedni wpis do pliku /etc/rc.conf. Spowoduje do zapisywanie ustawień przy każdym wyłączaniu sytemu oraz ich odtwarzanie.
File: /etc/rc.conf
DAEMONS=(...alsa...)

W przypadku, gdy ustawienia nie ładują się, dodaj jeden z poniższych wpisów do pliku /etc/rc.local:

File: /etc/rc.local
alsactl restore
File: /etc/rc.local
alsactl -f /var/lib/alsa/asound.state restore

[edytuj] Konfiguracja

  • Konfiguracja karty dźwiękowej:
# alsaconf
  • Aby umożliwić zwykłemu użytkownikowi korzystanie z karty dźwiękowej, dodaj użytkownika do grupy audio:
# gpasswd -a nazwa_użytkownika audio

Wyloguj i zaloguj ponownie użytkownika, aby wczytać nowe ustawienia grup.

  • Teraz z konta użytkownika możesz sterować głośnością poprzez mikser ALSA:
$ alsamixer

Aby ustawienia były zapamiętane (utworzony zostanie plik konfiguracyjny /etc/asound.state), wykonaj polecenie:

# alsactl store

Lub

# alsactl -f /var/lib/alsa/asound.state store
  • Aby sprawdzić, czy jest dźwięk:
$ aplay /usr/share/sounds/alsa/Front_Center.wav

Lub

$ speaker-test -c 2
  • Aby zrestartować ustawienia ALSA:
# /etc/rc.d/alsa restart

[edytuj] Wyłączenie głośniczka systemowego

Aby całkowicie wyłączyć dźwięki głośniczka systemowego należy dodać wpis w pliku /etc/modprobe.d/modprobe.conf, by moduł pcspkr nie były automatycznie ładowany wraz z uruchamianiem systemu operacyjnego:

File: /etc/modprobe.d/modprobe.conf
blacklist pcspkr

[edytuj] Equalizer

Linux pozwala ulepszyć sobie dźwięk według własnych wymogów. Zainstaluj pakiet alsaequal.
