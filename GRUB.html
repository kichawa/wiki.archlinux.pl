[edytuj] Wprowadzenie

GRUB jest programem rozruchowym, wykorzystywanym domyślnie przez Arch Linuksa. Jest bootloaderem, programem, który sprawuje kontrolę nad procesami zachodzącymi w trakcie uruchamiania jądra i systemu. Nadzoruje on również ładowanie modułów jądra, tak aby załadowane zostały tylko te, które są mu potrzebne. Nie tylko skraca to czas ładowania systemu, ale również pozwala na zachowanie cennych zasobów komputera podczas uruchamiania systemu z nośników wymiennych. Ładowanie systemu odbywa się automatycznie, zgodnie z ustawieniami zapisanymi w pliku konfiguracyjnym więc nie ma potrzeby ręcznego wpisywania komend. Jakkolwiek opcja taka również jest dostępna na wypadek błędu lub problemu z wczytaniem pliku konfiguracji.

[edytuj] Instalacja

GRUB jest domyślnie instalowany podczas instalacji Archa. Można go także zainstalować z repozytorium [core]:

# pacman -S grub

[edytuj] Konfiguracja

Plik konfiguracyjny znajduje się w /boot/grub/menu.lst, poniżej znajdziesz jego przykład (z oddzielną partycją /boot):

File: /boot/grub/menu.lst
# Config file for GRUB - The GNU GRand Unified Bootloader
# /boot/grub/menu.lst

# DEVICE NAME CONVERSIONS
#
#  Linux           Grub
# -------------------------
#  /dev/fd0        (fd0)
#  /dev/sda        (hd0)
#  /dev/sdb2       (hd1,1)
#  /dev/sda3       (hd0,2)
#

#  FRAMEBUFFER RESOLUTION SETTINGS
#     +-------------------------------------------------+
#          | 640x480    800x600    1024x768   1280x1024
#      ----+--------------------------------------------
#      256 | 0x301=769  0x303=771  0x305=773   0x307=775
#      32K | 0x310=784  0x313=787  0x316=790   0x319=793
#      64K | 0x311=785  0x314=788  0x317=791   0x31A=794
#      16M | 0x312=786  0x315=789  0x318=792   0x31B=795
#     +-------------------------------------------------+
#  for more details and different resolutions see
#  http://wiki.archlinux.org/index.php/GRUB#Framebuffer_Resolution

# general configuration:
timeout   5                                         # Czas oczekiwania (w sekundach) na Twój wybór,
                                                      po którym domyślny system operacyjny jest ładowany automatycznie.
default   0                                         # Która pozycja ma być uruchamiana domyślnie. 0 oznacza pierwszą, 1 drugą itd.
color light-blue/black light-cyan/blue

# boot sections follow
# each is implicitly numbered from 0 in the order of appearance below
#
# TIP: If you want a 1024x768 framebuffer, add "vga=773" to your kernel line.
#
#-*

# (0) Arch Linux
title  Arch Linux
root   (hd0,0)
kernel /vmlinuz26 root=/dev/sda3 ro
initrd /kernel26.img

# (1) Windows
#title Windows
#rootnoverify (hd0,0)
#makeactive
#chainloader +1
Wskazówka: Najważniejszym czynnikiem prowadzącym do zrozumienia programu GRUB jest pojęcie jego sposobu odnoszenia się do dysków i partycji. Partycja /dev/sda1 jest określana przez GRUB-a jako (hd0,0). Zauważ, że nawiasy po obu stronach hd0,0 są wymagane. Dyski twarde są liczone od zera, a nie od litery "a". Podobnie partycje także zaczynają się od zera, a nie od jedynki.

GRUB musi wiedzieć, gdzie jego pliki znajdują się w naszym systemie. Pliki konfiguracyjne zawsze znajdują się w katalogu /boot, który może być na specjalnej partycji. Jeśli nie znasz lokalizacji /boot, użyj polecenia grub, aby zlokalizować pliki GRUB-a.

Poniższy przykład jest dla systemów bez oddzielnej partycji /boot:

# grub> find /boot/grub/stage1

Poniższy przykład jest dla systemów z oddzielną partycją /boot:

$ # grub> find /grub/stage1
(hd0,0)

Ta wartość powinna być wpisana w wierszu z root w pliku konfiguracyjnym GRUB-a. Wpisz quit, aby wyjść z powłoki.

[edytuj] Dwa systemy - Linux & Windows

Dodaj na końcu pliku /boot/grub/menu.lst (zakładając, że na system Windows jest na pierwszej partycji pierwszego dysku):

File: /boot/grub/menu.lst
title Windows
rootnoverify (hd0,0)
makeactive
chainloader +1

Jeśli system Windows znajduje się na innym dysku, użyj polecenia map. To pozwoli Ci zainstalować system Windows, zakładając, że system jest na pierwszej partycji drugiego dysku:

File: /boot/grub/menu.lst
title Windows
map (hd0) (hd1)
map (hd1) (hd0)
rootnoverify (hd1,0)
makeactive
chainloader +1

[edytuj] Dwa systemy GNU/Linux

Przykład:

File: /boot/grub/menu.lst
title Inny Linux
root (hd0,2)
kernel /ścieżka/do/kernela root=/dev/sda3 ro
initrd /ścieżka/do/initrd

[edytuj] Dwa systemy GNU/Linux (Grub2)

Jeżeli druga dystrybucja Linuksa używa GRUB2 (na przykład Ubuntu w wersji 9.10 i wyższej), do pliku /boot/grub/menu.lst dodaj odpowiednie wpisy, jak ukazuje to przykład:

File: /boot/grub/menu.lst
# Inny Linuks korzystający z Grub2
title Ubuntu
root (hd0,2)
kernel /boot/grub/core.img

Wybór tego wpisu przy starcie uruchomi Menu Grub2 tej drugiej dystrybucji.

[edytuj] Chainloader i Configfile

Polecenia chainloader bądź configfile wykorzystywane są do uruchamiania innej dystrybucji Linuksa i zapewniają automatyczną konfigurację GRUB-a innych dystrybucji (na przykład Debian, Ubuntu czy openSUSE). Pozwala to na podział zarządzania własnym /boot/grub/menu.lst i opcjami uruchamiania systemów.

  • Polecenie chainloader ładuje inny bootloader (nie jądro innego systemu!); przydatne, jeśli inny bootloader jest instalowany w sektorze rozruchowym partycji.
  • Polecenie configfile pozwala na załadowanie określonego pliku konfiguracyjnego. Może być wykorzystywane do ładowania innej dystrybucji bez potrzeby instalacji kolejnego bootloadera.

[edytuj] Instalacjia bootloadera

[edytuj] Uwagi ogólne na temat instalacji bootloadera

GRUB może zostać zainstalowany z nośnika (na przykład LiveCD) lub z repozytoriów z uruchomionego Archa. Jego instalacja jest potrzebna, gdy:

  • Bootloader nie jest zainstalowany.
  • Kolejny system operacyjny nadpisał bootloadera Archa.
  • Bootloader nie działa, jak trzeba, przy czym powody tego są nieznane.

[edytuj] Instalacja w MBR

Przykład:

grub> setup (hd0)

[edytuj] Instalacja na partycji

Przykład:

grub> setup (hd0,0)

[edytuj] Alternatywna metoda instalacji

W celu instalacji bootloadera użyj polecenia grub-install. Przykładowo, instalacja jego w MBR wyglądać będzie następująco:

# grub-install /dev/sda

[edytuj] Przydatne wskazówki

[edytuj] Graficzny wygląd ładowania systemu

W tym pomoże Tobie grub-gfx:

  • Zrób kopię zapasową pliku /boot/grub/menu.lst:
# cp /boot/grub/menu.lst /boot/grub/menu.lst.bak

Następnie zainstaluj grub-gfx z AUR, po czym sprawdź, czy wpisy w pliku /boot/grub/menu.lst są prawidłowe z utworzoną wcześniej kopią zapasową.

  • Do pliku /boot/grub/menu.lst dodaj wiersz z splashimage, na przykład:
File: /boot/grub/menu.lst
# general configuration:
timeout   5
default   0
color light-blue/black light-cyan/blue
splashimage /boot/grub/arch.xpm.gz
  • Na koniec przeinstaluj bootloadera. Użyj wspomnianego wyżej polecenia grub-install. Poniżej masz przykład:
# grub-install /dev/sda

[edytuj] Rozdzielczość bufora ramki

Otóż, jak zapewne już wiesz, możesz zastosować rozdzielczość podaną w /boot/grub/menu.lst, jednakże w przypadku posiadania szerokiego ekranu LCD masz mozliwość ustawienia pełnej rozdzielczości rodzimej.

Wskazówka: Na stronie Wikipedii znajduje się lista rozdzielczości bufora ramki. Jednakże, nie na każdą rozdzielczość może pozwalać twoja konfiguracja sprzętowa, dlatego też zalecane jest skorzystanie z jednego z poniższych narzędzi - znacząco ułatwiających pracę.

Vbetest

  1. Zainstaluj pakiet lrmi z repozytorium [community], który zawiera narzędzie vbetest.
  2. Uruchom vbetest jako root.
  3. Zanotuj numer w nawiasie [ ], odpowiadający żądanej rozdzielczości.
  4. Naciśnij 'q', aby zakończyć.
  5. Dodaj liczbę 512 do wartości odpowiadającej rozdzielczości.
  6. Przykład:

Jeżeli zdecydowałeś się na rozdzielczość 1440x900, dzięki vbetest otrzymałeś zapewne: 

[357] 1440x900 (8:8:8)

W tym momencie musisz dodać 357 i 512, co daje wynik 869. Tak więc do pliku /boot/grub/menu.lst dodajesz taki oto wpis:

File: /boot/grub/menu.lst
kernel /vmlinuz26 root=/dev/sda1 ro vga=869

Hwinfo

  1. Zainstaluj pakiet hwinfo z [Arch User Repository|AUR].
  2. Uruchom hwinfo --framebuffer jako root.
  3. Zanotuj kod odpowiadający żądanej rozdzielczości.
  4. Użyj tego kodu z przedrostkiem 0x w opcji krenela vga w pliku /boot/grub/menu.lst.

Przykład: 

Mode 0x0364: 1440x900 (+1440), 8 bits
Mode 0x0365: 1440x900 (+5760), 24 bits
File: /boot/grub/menu.lst
kernel /vmlinuz26 root=/dev/sda1 ro vga=0x0365

[edytuj] GRUB sam rozpoznaje wartość vga

Jest prosty sposób na to, aby GRUB ustalał wrtość vga odpowiadającą danej rozdzielczości.

Dodaj do pliku /boot/grub/menu.lst pogrubiony na poniższym przykładzie tekst:

File: /boot/grub/menu.lst
kernel /vmlinuz26 root=/dev/sda1 ro vga=ask

Zrestartuj Archa. Po ponownym uruchomieniu GRUB pokaże tobie listę kodów odpowiadającym rozdzielczościom, z których wybierz jeden. Zastąp ask w linii z kernelem na wartość, jaką wybrałeś. Na przykład [369] 1680x1050x32:

File: /boot/grub/menu.lst
kernel /vmlinuz26 root=/dev/sda1 ro vga=873

[edytuj] Używanie etykiet partycji

Jeśli zmieniasz lub zamierzasz zmienić rozmiary partycji, rozważ zdefiniowanie ich za pomocą etykiet. W tym użyj polecenia, jak na przykładzie:

# e2label </dev/dysk/partycja> nazwa_etykiety

Nazwy etykiety może wynosić do 16 znaków. Następnie zdefiniuj ją w menu.lst:

File: /boot/grub/menu.lst
kernel /boot/vmlinuz26 root=/dev/disk/by-label/Arch_Linux ro

[edytuj] Ochrona hasłem

Jeśli chcesz, możesz włączyć ochronę hasłem w pliku konfiguracyjnym GRUB.

Po pierwsze, ustal swoje hasło i zaszyfruj je:

$ # grub-md5-crypt
Password:
Retype password:
$1$ZOGor$GABXUQ/hnzns/d5JYqqjw

Następnie dodaj hasło na początku pliku konfiguracyjnego GRUB:

File: /boot/grub/menu.lst
# general configuration
timeout   5
default   0
color light-blue/black light-cyan/blue
password --md5 $1$ZOGor$GABXUQ/hnzns/d5JYqqjw

Następnie dla każdego systemu operacyjnego, które chcesz ochronić, dodaj polecenie lock:

File: /boot/grub/menu.lst
# (0) Arch Linux
title  Arch Linux
lock
root   (hd0,1)
kernel /boot/vmlinuz26 root=/dev/disk/by-label/Arch_Linux ro
initrd /boot/kernel26.img

[edytuj] Wyłączanie zbędnych modułów

Jeżeli chcesz, aby dany moduł nie ładował się wraz z systemem, dodaj odpowiedni wpis (modprobe.blacklist=...)do linii z kernelem, na przykład:

File: /boot/grub/menu.lst
# (0) Arch Linux
title  Arch Linux
root   (hd0,1)
kernel /boot/vmlinuz26 root=/dev/disk/by-label/Arch_Linux ro modprobe.blacklist=moduł1,moduł2,moduł3
initrd /boot/kernel26.img
