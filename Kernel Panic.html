[edytuj] Wprowadzenie

Kernel panic jest komunikatem wyświetlanym przez system operacyjny po wykryciu wewnętrznego błędu jądra systemu, którego sam nie jest w stanie obsłużyć. Często komunikat ten dostarcza również niezbędnych do odkrycia przyczyny błędu informacji lub danych potrzebnych do debugowania przez programistów pracujących nad systemem.

[edytuj] Rozwiązywanie problemów

Aby ułatwić rozwiązywanie problemów,sprawdź, czy jądro jest w trybie cichym. Jeśli tak, usuń quiet z konfiguracji bootloadera (np. /boot/grub/menu.lst). Upewnij się także, że system nie jest wadliwy. W tym celu możesz uruchomić memtest z płyty instalacyjnej Arch Linuksa.

[edytuj] Opcja 1: Sprawdź konfigurację bootloadera

Inną możliwością jest błąd w konfiguracji bootloadera (np. /boot/grub/menu.lst). Na przykład, partycjonowanie dysków twardych może zmienić porządek partycji, bądź wkradły się w pliku jakieś literówki. Nawet znak spacji postawiony w złym miejscu powoduje Kernel Panic. Dlatego ważne jest, abyś przypomniał sobie, czy ostatnio nie wykonałeś jakiejś czynności, która mogłaby wpłynąć na bootloader.

[edytuj] Opcja 2: Ponowna instalacja jądra

Ponowna instalacja jądra jest prawdopodobnie najlepszym rozwiązaniem.

[edytuj] Start z płyty instalacyjnej

Pierwszym krokiem jest uruchamianie płyty instalacyjnej. Po uruchomieniu, zaloguj się na konto root.

[edytuj] Montowanie partycji

W tym punkcie znajdujesz się w minimalnym, ale funkcjonalnym środowisku z podstawowymi narzędziami. Musisz teraz zamontować normalny dysk root do /mnt:

# mount /dev/sdXY /mnt

Jeśli używasz starszej konfiguracji IDE, użyj polecenia:

# mount /dev/hdXY /mnt

Jeśli używasz osobnej partycji rozruchowej, nie zapomnij jej zamontować:

# mount /dev/sdXZ /mnt/boot

[edytuj] Kopia plików systemowych (opcjonalnie)

To jest dobry moment, aby ważniejsze pliki systemowe w formie kopii na inną partycję czy pamięć przenośną. Mogą przydać się później. Oczywiście nie kopiuj wszystkich plików - tylko te, które mogą zmienić się podczas poniższych operacji.

[edytuj] Konfiguracja środowiska chroot

Jądro systemu używa ramdysku do ustanowienia swojego środowiska. Po reinstalacji kernela ramdysk zostanie ponownie wygenerowany przez Mkinitcpio. Niestety, narzędzie to nie wykryje wszystkich modułów, niezbędnych do uruchomienia systemu. Dlatego też musisz wykonać poniższe polecenia:

# mount -t proc none /mnt/proc
# mount -t sysfs none /mnt/sys
# mount --bind /dev /mnt/dev

[edytuj] Chroot i przywrócenie poprzedniej wersji jądra

Chrootujemy się przy pomocy polecenia:

# chroot /mnt

Następnie sprawdź, że posiadasz jeszcze poprzednie wersje kernela w pamięci podręcznej Pacmana. Jeśli tak nie jest, musisz we własnym zakresie zdobyć pakiet (sprawdź płytę instalacyjną, pakiet z kernelem powinien być w addons/core-pkgs/). Jednak załóżmy, że nie czyściłeś pamięci Pacmana od jakiegoś czasu. Najpierw musisz uzyskać szczegóły pakietów jądra:

# cd /var/cache/pacman/pkg
# find kernel*

Teraz za pomocą poniższego polecenia zainstaluj poprzednią wersję kernela:

# pacman -U /var/cache/pacman/pkg/kernel26-2.6.XX.XX-X.pkg.tar.xz

[edytuj] Restart systemu

Teraz zrestartuj system i sprawdź czy wszystko już jest w porządku.
