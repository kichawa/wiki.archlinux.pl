[edytuj] Wprowadzenie

U niektórych użytkowników Arch Linuksa rozruch procesu Udev trwa bardzo długo. Najnowsze wersje udev nie tylko zawierają nowe pliki binarne, ale także zoptymalizowane skrypty. Skrypt /lib/udev/load-modules.sh zawiera kod z czarną listą niektórych modułów. Skrypt jest dobrze napisany, ale ponieważ wykorzystywany jest do uruchamiania każdego modułu, może spowalniać proces uruchamiania. Dlatego warto wykorzystać ten artykuł aby zoptymalizować mechanizm udev.

Uwaga!
Poniższe czynności mogą uszkodzić Twój system. Przed wykonaniem jakiejkolwiek operacji z tego artykułu warto zrobic kopię bezpieczeństwa swojego systemu.

[edytuj] Opcja 1

  • Zrób kopię zapasową pliku /lib/udev/load-modules.sh, na przykład o nazwie /lib/udev/load-modules.old:
# cp -i -p /lib/udev/load-modules.sh /lib/udev/load-modules.sh.old
  • Zastąp zawartość /lib/udev/load-modules.sh przy użyciu polecenia echo:
# echo '#!/bin/bash' > /lib/udev/load-modules.sh && echo '/sbin/modprobe $1' >> /lib/udev/load-modules.sh
  • Możesz również sprawdzić zawartość /lib/udev/load-modules.sh, która powinna wyglądać: 
#!/bin/bash
/sbin/modprobe $1
  • W razie problemów możesz przywrócić kopię /lib/udev/load-modules.sh za pomocą polecenia:
# mv -i /lib/udev/load-modules.sh.old /lib/udev/load-modules.sh

[edytuj] Opcja 2

  • Zrób kopię zapasową pliku /lib/udev/load-modules.sh, na przykład o nazwie /lib/udev/load-modules.old:
# cp -i -p /lib/udev/load-modules.sh /lib/udev/load-modules.old
  • Zastąp wszystkie wystąpienia /lib/udev/load-modules.sh na /sbin/modprobe w /etc/udev/*.rules używając narzędzia sed:
# sed -i "s#/lib/udev/load-modules.sh#/sbin/modprobe#g" /etc/udev/rules.d/*.rules
  • W razie problemów przywróć wszystkie wystąpienia:
# sed -i "s#/sbin/modprobe#/lib/udev/load-modules.sh#g" /etc/udev/rules.d/*.rules
  • W razie problemów możesz przywrócić kopię /lib/udev/load-modules.sh:
# mv -i /lib/udev/load-modules.sh.old /lib/udev/load-modules.sh

[edytuj] Opcja 3

  • Zrób kopię zapasową pliku /lib/udev/load-modules.sh, na przykład o nazwie /lib/udev/load-modules.old:
# cp -i -p /lib/udev/load-modules.sh /lib/udev/load-modules.old
  • Utwórz link symboliczny pliku /lib/udev/load-modules.sh do folderu /sbin/modprobe:
# ln -sf /sbin/modprobe /lib/udev/load-modules.sh
  • W razie problemów możesz przywrócić kopię /lib/udev/load-modules.sh za pomocą polecenia:
# mv -i /lib/udev/load-modules.sh.old /lib/udev/load-modules.sh
  • Ponieważ skrypty, pliki binarne i nazewnictwo mogą zmieniać się z czasem, dlatego też zaleca się nie aktualizować udev:
# pacman -Syu --ignore udev
  • Możesz również dodać udev do opcji #IgnorePkg w pliku /etc/pacman.conf: 
IgnorePkg = udev

W trakcie aktualizacji możesz otrzymać poniższy komunikat. Oczywiście nie aktualizuj pakietu udev: 

warning: udev: ignoring package upgrade (161-1 => 161-2)
:: udev is in IgnorePkg/IgnoreGroup. Install anyway? [Y/n] n

Jeżeli zamierzasz zaktualizować udev, przywróć kopię oryginalną /lib/udev/load-modules.sh.

  • W razie problemów możesz przywrócić kopię /lib/udev/load-modules.old:
# mv -i /lib/udev/load-modules.sh.old /lib/udev/load-modules.sh

[edytuj] Konfiguracja modułów

W tym miejscu opisane są sposoby wyłączenia modułów, które są według ciebie zbędne.

[edytuj] /etc/modprobe.conf

  • W pliku /etc/modprobe.d/modprobe.conf wyłączasz moduły w poniższy sposób:
File: /etc/modprobe.d/modprobe.conf
blacklist MODULE_1
blacklist MODULE_2
  • Drugi sposób:
File: /etc/modprobe.d/modprobe.conf
install MODULE_1 /bin/false
install MODULE_2 /bin/false

[edytuj] /etc/rc.local

  • Dodaj odpowiedni wpis oznaczający dodanie modułu do czarnej listy w pliku /etc/rc.local: 
rmmod nazwa_modułu_1
rmmod nazwa_modułu_2
  • Ewentualnie: 
modprobe -r nazwa_modułu_1
modprobe -r nazwa_modułu_2

[edytuj] Wyłączanie zbędnych modułów w pliku bootloadera

Jeżeli chcesz, aby dany moduł nie ładował się wraz z systemem, dodaj odpowiedni wpis (modprobe.blacklist=...)do linii z kernelem, na przykład:

File: /boot/grub/grub.cfg
 # (0) Arch Linux
 menuentry "Arch Linux" {
 set root=(hd0,1)
 linux /boot/vmlinuz26 root=/dev/sda1 ro modprobe.blacklist=moduł1,moduł2,moduł3
 initrd /boot/kernel26.img
 }

[edytuj] Zależności

Poniższe polecenia pomogą określić zależności modułów.

  • Wynik polecenia pokaże, jakie moduły są obecnie załadowane.
$ lsmod
  • Użyj polcenia, aby zdobyć informacje o module MODULE_1:
$ modinfo MODULE_1
  • Użyj polcenia, aby zdobyć informacje o module MODULE_2 ( z informacją o zależnościach):
$ modprobe --show-depends MODULE_2
